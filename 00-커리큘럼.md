# 세종청년센터 — 바이브 코딩으로 수익나는 앱 만들기

## 강의 개요

- **일정**: 2026-04-27(월) ~ 2026-04-30(목), 총 4일
- **시간**: 매일 18:00 ~ 22:00 (4시간 × 4일 = 16시간)
- **장소**: 세종청년센터
- **강사**: 본인
- **주제**: 바이브 코딩으로 수익나는 앱 만들기

## 수강생 정보

- **타깃**: 비전공자 (코드 작성 경험 거의 없음 전제)
- **예상 인원**: 20명 이상
- **사전 지식 수준**: ChatGPT 한두 번 써본 수준 가정
- **결제 환경**: 유료 AI 미결제 다수 → **모든 도구는 무료 한도 안에서 굴러가야 함**
- **지참 장비**: 노트북 (Mac/Windows 무관), 충전기, 마우스 권장

## 강의 목표 (현실적 재정의)

> "4일 만에 수익 내는 상용 앱 완성"은 비현실적이다. 진짜 목표는:
> 1. AI가 어디까지 왔는지 **체감**한다
> 2. 본인이 다시 시도할 수 있는 **워크플로우와 도구 목록**을 손에 쥔다
> 3. 본인 아이디어 기반의 **데모 1개**를 직접 띄워본다
> 4. 수익화의 **구조**를 이해한다 (실제 입금까진 사업자 등록 등 별도 절차)

## 사용 툴 (전부 무료 한도 내 사용 가능)

- **메인 앱 빌더**: Lovable (브라우저 기반, 비전공자 진입장벽 가장 낮음)
- **백업 빌더**: v0(Vercel), Bolt(StackBlitz) — Lovable 메시지 소진 시 대안
- **챗봇**: ChatGPT 무료, Claude 무료, Gemini, Perplexity (요구사항 정리·카피라이팅·트러블슈팅)
- **이미지**: Nano Banana(Gemini), Bing Image Creator(DALL-E 3), Ideogram (로고·아이콘)
- **음악·음성**: Suno, ElevenLabs (Day 1 체험용)
- **영상**: Runway, Pika, Veo (Day 1 체험용)
- **백엔드**: Supabase 무료 (인증 / DB / 스토리지)
- **배포**: Vercel 무료
- **결제(시연)**: Stripe Payment Links, 토스페이먼츠 테스트 모드, Gumroad

## 4일 커리큘럼 설계

### Day 1 — 2026-04-27 (월) | AI 도구 만물상 투어

- **목표**: AI가 어디까지 왔는지 직접 만져보고 "우와" 체험
- **블록**: 텍스트 LLM → 이미지 → 음악·음성 → 영상 → 노코드 앱 빌더
- **숙제**: 본인 앱 아이디어 1개 가져오기

### Day 2 — 2026-04-28 (화) | 앱 컨셉 잡고 화면 만들기

- **목표**: 각자 앱 컨셉 확정 + Lovable로 첫 화면 띄우기
- **핵심 개념**: 좋은 프롬프트 = 좋은 PRD, 디자인 일관성 유지
- **실습**: ChatGPT로 PRD 만들기 → Lovable 프로젝트 생성 → 메인+서브 화면

### Day 3 — 2026-04-29 (수) | 데이터·인증·배포

- **목표**: 정적 화면을 "진짜 앱"으로 (로그인, 데이터 저장, 인터넷에 띄우기)
- **핵심 개념**: 프론트/백엔드 분리, BaaS 개념, 환경 변수, 배포
- **실습**: Supabase 연결 → 회원가입/로그인 → CRUD 한 사이클 → Vercel 배포

### Day 4 — 2026-04-30 (목) | 수익화 + 발표

- **목표**: 수익화 구조 이해 + 본인 데모 발표
- **핵심 개념**: PG vs 결제링크, 구독 vs 일회성, 랜딩페이지·SEO
- **실습**: 결제 시연 → 랜딩페이지 → 공유 링크 → 1인 1~2분 발표

## 운영 원칙

- **시연 → 따라하기 → 짝꿍 도움** 구조 (1:1 코칭 불가능, 옆사람 도움 적극 유도)
- **회사 동료 보조 인력 동행** → Day 3 트러블슈팅 분산 담당. 사전에 역할 분담 협의 필요
- 진도 못 따라오는 사람 다수 발생 전제 → "끝까지 못 만들어도 괜찮다, 워크플로우만 챙겨가라" 반복 강조
- 무료 한도 소진 대비 백업 도구 항상 1개 이상 준비
- 발표는 "완성도" 평가 아니라 "본인이 무엇을 시도했는가" 공유에 가깝게

## 최종 결과물

- 본인 아이디어 기반 웹앱 1개 (배포된 URL 포함이 이상적, 안 되면 화면이라도)
- 로그인/데이터 저장 흐름 이해
- 결제 연동 구조 이해 (실제 입금까진 사업자등록 별도)
- 강의 후 혼자서 다음 시도를 할 수 있는 도구 리스트 + 워크플로우
