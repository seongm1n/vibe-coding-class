## 핵심 개념

- **프롬프트 = 지시서**: AI는 무한히 시킬 수 있는 인턴이지만, **명확하게 시켜야** 결과가 나온다
- **모든 AI는 다 다르다**: 같은 질문도 ChatGPT/Claude/Gemini 답이 다르다 → 비교 감각이 중요
- **무료 한도 안에서 살아남기**: 도구마다 무료 한도와 강점·약점이 있다
- **결과물보다 워크플로우**: "오늘 만든 것"보다 "내일 혼자 다시 할 수 있는가"가 중요

## 블록별 실습 내용

### ① 텍스트 LLM 비교 (30분)

- 같은 프롬프트를 4개 챗봇에 동시에 던져보기
  - 예시 프롬프트: "세종에 사는 30대 직장인이 점심에 갈만한 식당 추천 컨셉 앱 아이디어 5개 줘"
- 비교 포인트: 답변 길이, 구체성, 한국어 자연스러움, 최신성
- 실습: 본인 관심 주제로 직접 4곳에 같은 질문 던지고 차이 느껴보기

### ② 이미지 생성 (30분)

- Nano Banana (Gemini): 무료, 한국어 프롬프트 잘 먹음
- Bing Image Creator: DALL-E 3 무료, MS 계정만 있으면 됨
- Ideogram: 텍스트가 들어간 이미지에 강함 (로고에 좋음)
- 실습: 본인 앱(아직 미정이어도 OK) 로고 1개 만들기 → Day 2에 사용

### ③ 음악·음성 (30분)

- Suno: 가사+장르 입력하면 노래 1곡 생성 (무료 일 10크레딧)
- ElevenLabs: 텍스트 → 음성 합성, 한국어 가능
- 실습: 본인 앱 홍보 BGM 한 줄 + 안내 음성 1문장 만들어보기
- "우와 포인트": 진짜 가수처럼 노래 부르는 데서 분위기 한 번 띄우기

### ④ 영상 생성 (30분)

- Runway / Pika / Veo: 무료 한도가 짠 편 → **강사 시연 위주**, 수강생은 1명당 1개 시도
- 실습: 짧은 5초짜리 영상 1개 생성 (텍스트 → 비디오)
- 한계도 함께 설명: 렌더 시간, 무료 한도, 결과 일관성 부족 → "지금은 체험"이라는 점 강조

### ⑤ 노코드 앱 빌더 미리보기 (40분)

- Lovable 메인 시연 (강사가 라이브로 1개 만들어 보이기)
  - 프롬프트 → 화면 → 수정 → 배포까지 풀 사이클
- v0 (Vercel) / Bolt (StackBlitz) 짧게 비교: 강점·약점·무료 한도
- 수강생은 보고만 있어도 OK — Day 2에 직접 만든다

## 시연 코드 / 예제

- **WiFi 부하**: 영상 생성·노코드 빌더에서 동시 트래픽 몰림 주의 → 순차 진행
- **무료 한도 소진**: Suno 10크레딧, Runway 영상 1~2개 등 빠르게 소진됨 → "체험은 1번만, 욕심내지 말기" 안내
- **계정 안 만들어 온 사람** 대비: 가장 빠른 가입 동선 (예: Google OAuth로 통일) 미리 정리

## 예상 질문 & 대응

- **"유료로 안 사도 충분히 쓸 수 있나요?"** → 수익이 나기 전까진 무료로 충분. 본격적으로 만들 때 도구 1~2개만 유료 전환 권장
- **"어떤 LLM이 제일 좋아요?"** → 용도별로 다름. 길게 쓸 글은 Claude, 빠른 답변과 검색은 ChatGPT/Perplexity, 무료에 한국어는 Gemini
- **"이미지/영상 저작권은요?"** → 도구마다 약관 다름. 상업 사용 가능 도구 미리 정리해서 답변
- **"이거 할 줄 알면 회사에서 잘릴까요?"** → AI는 사람을 대체하기보다 "AI 잘 쓰는 사람"이 다른 사람을 대체. 이번 강의가 그 출발점

## 숙제
