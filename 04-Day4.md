## 핵심 개념

### 수익화 모델 4가지

1. **일회성 결제**: 디지털 상품(템플릿, 강의), 한 번 받고 끝
2. **구독(SaaS)**: 매달/매년 정기 결제. 가장 안정적이지만 가입 진입장벽 높음
3. **광고**: 트래픽이 일정 수준 이상이어야 의미 있음 (초기엔 비추)
4. **수수료/중개**: 두 사용자 사이에서 거래가 일어날 때 수수료. 양면시장 구축이 어려움

### 결제 도구 비교

| 도구 | 진입장벽 | 사업자 필요 | 한국 사용자 친화 | 추천 시나리오 |
|------|---------|------------|-----------------|--------------|
| Stripe Payment Links | 낮음 | 미국 법인 또는 우회 | 낮음 | 해외 대상 |
| 토스페이먼츠 | 중간 | 사업자등록·통신판매업 신고 | 매우 높음 | 한국 대상 본격 |
| Gumroad | 매우 낮음 | 불필요 | 중간 (PayPal/카드) | 디지털 상품 빠른 판매 |
| Lemon Squeezy | 낮음 | 불필요 (대신 처리) | 중간 | 글로벌 SaaS 시작 |

### "수익이 나려면" 진짜 필요한 것

- 결제 모듈은 **가장 마지막 단계**일 뿐
- 진짜 어려운 건: 사람들이 **이 앱을 발견 → 가입 → 돈 낼 가치를 느끼게** 하는 것
- 그래서 랜딩페이지 / SEO / 공유 메시지가 결제 모듈만큼 중요

### 랜딩페이지의 역할

- 앱 자체가 아니라 **앱을 설명하는 1페이지**
- 구성: 한 줄 헤드라인 → 문제 정의 → 해결 방법 → 스크린샷 → CTA(가입/결제 버튼)
- 도메인 첫 페이지를 랜딩페이지로 두고, 내부에서 앱으로 연결

## 블록별 실습 내용

### ① 수익화 구조 강의 (45분)

- 4가지 모델 설명 + 각각 실제 예시 (한국 인디 메이커 사례 2~3개)
- "본인 앱은 어디에 가까운가?" 짝꿍과 1분 토론
- 사업자등록 / 통신판매업 신고 흐름 짧게 설명 (1년 매출 4800만 미만이면 면세사업자 가능 등)

### ② 결제 도구 체험·시연 (60분)

- 강사 시연 (30분):
  - Gumroad에 "데모 디지털 상품" 1개 등록 → 결제 링크 생성 → 본인 앱에 버튼으로 붙이기
  - 토스페이먼츠 테스트 모드 결제창 띄우는 시연 (실 결제 X)
  - Stripe Payment Links 가입까지만 (실 사용은 해외 대상일 때 의미)
- 수강생 실습 (30분): 본인 앱에 결제 버튼 1개 (어떤 도구든) 추가
- "오늘 진짜 돈 받는 게 아니라, 구조를 손으로 만져본다는 게 핵심"

### ③ 랜딩페이지 + SEO + 도메인 (50분)

- 랜딩페이지: Lovable에 "위 앱을 소개하는 1페이지 랜딩페이지 만들어줘. 헤드라인 / 문제 / 해결 / 스크린샷 / 가입 버튼 구성" 프롬프트
- SEO 기초:
  - 페이지 제목(title), 설명(meta description) 무엇을 적어야 하는가
  - 검색어 1~2개를 헤드라인에 넣기
- 도메인 연결 (선택):
  - 무료: Vercel 기본 도메인 그대로 사용
  - 유료: 가비아/Cloudflare에서 .com 도메인 ($10~15/년) 사서 Vercel 연결
  - 시간 부족 시 도메인은 데모만

### ④ 발표 (40분)

- 1인 1~2분 (20명 × 2분 = 40분)
- 발표 형식: ① 누구를 위한 앱인지 ② 4일간 무엇을 시도했는지 ③ 다음에 뭘 해보고 싶은지
- "완성도" 평가 X, "시도" 공유에 가깝게
- 본인 앱 URL을 단톡방/패들렛 같은 데 다같이 공유

### ⑤ 마무리 (20분)

- 자료 일괄 공유: 강의 슬라이드, 프롬프트 템플릿, 추천 도구 리스트
- 강의 후 학습 가이드:
  - 더 깊게 가고 싶다면: Cursor 14일 트라이얼, YouTube 채널 추천 1~2개
  - 운영하고 싶다면: 사업자등록 → 토스페이먼츠 가입 흐름
  - 막혔을 때: 본인 PRD 다시 보기, ChatGPT에 에러 메시지 그대로 붙이기
- 수료증 / 출석 등 행정 처리 (센터 측 요구사항)
- 피드백 폼 안내

  - 통신판매업 신고 (정부24, 무료~소액 수수료)
  - 토스페이먼츠 또는 KG이니시스 가입

## 예상 질문 & 대응

- **"사업자등록 안 하고 돈 받을 수 있어요?"** → 일정 금액 이하 비정기 수익은 가능하지만 권장 X. 정식 운영하려면 등록 필수
- **"부가세는요?"** → 연 매출 4800만 미만이면 간이과세 가능. 자세한 건 세무사 상담 권장
- **"앱이 잘 안 되면 환불해줘야 하나요?"** → 전자상거래법상 디지털 상품도 환불 정책 명시 의무. 약관에 명확히 적기
- **"광고로 돈 벌고 싶어요"** → 월 방문자 1만명 정도는 되어야 의미 있음. 초기엔 결제 모델이 더 빠른 검증
- **"앞으로 뭘 더 배우면 좋아요?"** → ① 본인 앱 사용자 1명 만들기 ② 그 사용자에게 인터뷰 ③ 그 다음 기능 정하기. 추가 학습보다 실사용자 1명이 훨씬 큰 도약
