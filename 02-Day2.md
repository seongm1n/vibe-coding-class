## 핵심 개념

### PRD = 프롬프트의 본체

- 비전공자도 PRD를 모르는 게 아니라 **이름만 모르는 것**
- "누가, 왜, 무엇을, 어떻게" 4개만 정리해도 충분한 PRD
- AI에게 시킬 때 이걸 빼먹으면 결과가 일관성 없게 나옴

### 프롬프트 작성의 3가지 원칙

1. **컨텍스트**: "나는 30명 규모 헬스장을 운영하는 사장이다"
2. **목표**: "회원이 출석 체크를 셀프로 할 수 있는 화면이 필요하다"
3. **제약**: "모바일에서 잘 보여야 하고, 글자는 크게, 한국어로"

### Lovable 메시지 아껴쓰기

- 무료 한도(매일 5메시지 전후) → **한 번에 많이 시키고 결과 보고 다음 메시지** 패턴
- "조금씩 고치기"보다 "한꺼번에 큰 변화 요청"이 효율적
- 화면 단위로 끊어서 진행 (한 메시지에 한 화면 완성 목표)

## 블록별 실습 내용

### ① PRD 사고법 (45분)

- 강사 시연: ChatGPT에 다음 프롬프트 던지기
  ```
  나는 [컨셉]을 만들고 싶다.
  비개발자도 이해할 수 있는 1페이지 PRD를 만들어줘.
  - 타겟 사용자
  - 해결하려는 문제
  - 핵심 기능 3개
  - 가장 중요한 화면 2~3개의 구성 요소
  ```
- 출력된 PRD를 함께 읽고 "이 정도면 AI가 화면 만들 수 있겠다" 감 잡기

### ② 본인 앱 컨셉 정의서 (60분)

- 어제 숙제로 가져온 아이디어로 위 프롬프트 본인 버전 작성
- 짝꿍 1명에게 PRD 보여주고 "이거 보고 뭐 만들지 상상돼?" 검증
- 막히는 사람 → 강사가 미리 준비한 **5가지 보편 아이디어** 제공
  - 예: 동네 식당 영업시간 모음, 운동 기록 일기, 가계부, 독서 메모, 공구 모임 정리

### ③ Lovable 첫 화면 (80분)

- 강사 시연 (15분):
  - Lovable 새 프로젝트 → PRD 통째로 붙여넣기 → 결과 보기
  - 색상 / 폰트 / 레이아웃 한 메시지로 동시에 수정 요청하는 법
  - 막혔을 때 → "이전 화면으로 돌아가기" 사용법
- 수강생 실습 (60분): 본인 PRD로 첫 화면 띄우기
- 진도 빠른 사람 → 서브 화면(상세 페이지 / 입력 폼) 추가
- 진도 못 따라오는 사람 → 강사 준비한 PRD 그대로 사용

### ④ 화면 다듬기 (30분)

- 어제 만든 로고 / 이미지 적용
- 색상·폰트 통일
- 모바일에서 어떻게 보이는지 미리보기로 확인

## 시연 코드 / 예제

- 강사용 데모 PRD 1개 (Day 1에 정해둔 컨셉으로)
- "막힌 사람용" 백업 PRD 5개 미리 준비
- Lovable 프롬프트 템플릿 1장 출력해서 배포

## 강사용 진행 메모

- **PRD 단계에서 시간 다 쓰는 사람** 주의 → 60분 지나면 강제로 Lovable 단계로 넘기기 ("PRD는 만들면서 계속 고치면 된다")
- **Lovable 무료 한도 소진** → v0 또는 Bolt 백업으로 즉시 전환 안내
- **"내 화면이 이상하게 나와요"** → 대부분 PRD가 모호한 게 원인. PRD 다시 보자고 유도
- **20명 동시 사용** → Lovable 서버 응답 느려질 수 있음 → 휴식 시간을 약간 유연하게

## 예상 질문 & 대응

- **"내 아이디어가 너무 작은 것 같아요"** → 작을수록 4일 안에 완성됨. 큰 건 작은 것부터 쪼개는 게 정석
- **"Lovable 결과가 마음에 안 들어요"** → PRD에 없던 기준을 머릿속에만 갖고 있는 경우 다수. 그 기준을 PRD에 추가
- **"코드를 봐야 하나요?"** → 오늘은 No. 내일(Day 3)부터 코드를 살짝 본다 (보기만 함)
- **"이대로 앱스토어에 올릴 수 있나요?"** → 웹앱이라 앱스토어 X, 인터넷 주소(URL)로 공유 가능. 모바일 앱은 별도 과정
