## 핵심 개념

### PRD = 프롬프트의 본체

- 비전공자도 PRD를 모르는 게 아니라 **이름만 모르는 것**
- "누가, 왜, 무엇을, 어떻게" 4개만 정리해도 충분한 PRD
- AI에게 시킬 때 이걸 빼먹으면 결과가 일관성 없게 나옴

### 프롬프트 작성의 3가지 원칙

1. **컨텍스트**: "나는 30명 규모 헬스장을 운영하는 사장이다"
2. **목표**: "회원이 출석 체크를 셀프로 할 수 있는 화면이 필요하다"
3. **제약**: "모바일에서 잘 보여야 하고, 글자는 크게, 한국어로"

### Lovable 메시지 아껴쓰기

- 무료 한도(매일 5메시지 전후) → **한 번에 많이 시키고 결과 보고 다음 메시지** 패턴
- "조금씩 고치기"보다 "한꺼번에 큰 변화 요청"이 효율적
- 화면 단위로 끊어서 진행 (한 메시지에 한 화면 완성 목표)

## 블록별 실습 내용

### ⓪ 아이디어 점검 + 늦은 분 따라잡기 (25분, 워밍업)

**왜 이 블록이 강의 맨 앞에 있나**: 어제 늦게 도착하시는 분들 / 아이디어를 못 가져오신 분들 / 너무 큰 아이디어를 가져오신 분들을 한 번에 받쳐주는 시간. 이 25분을 잘 쓰면 뒤 블록이 훨씬 매끄러움.

- **Day 1 핵심 3줄 요약** (2분) — 늦게 오신 분들이 자연스럽게 따라옴
- **아이디어 발굴 5질문** (10분, 아이디어 없는 사람용)
  1. 본인 일과 5분 단위로 적어 보기
  2. 가족·동료의 잦은 불평
  3. 폰에 깔린 안 쓰는 앱
  4. 자주 쓰는 엑셀·노트
  5. 부모님·할머니가 못 쓰시는 앱
- **아이디어 복잡도 가늠** (10분, 핵심 학습)
  - 🟢 청신호 5개 / 🔴 Red Flag 8가지 / 🟡 헷갈리는 함정 9개
  - 비전공자가 가장 많이 헷갈리는 영역: "공유"·"알림"·"결제"·"검색"·"추천" 같은 단어
  - **단순 버전 vs 풀 버전 차이를 표로 명확히**
- **아이디어 단순화 3가지 자르기 전략** (3분)
  1. 사용자 1명 버전부터
  2. 가장 자주 쓰는 1개 기능만
  3. 수동으로 풀 수 있는 건 수동으로
- **Before/After 단순화 예시 3개**: 슬랙 / 우버 / 배민 클론 → 4일짜리

> 강사 멘트: "여기서 잘라두면 PRD·Lovable 단계에서 헛수고를 안 합니다. 5분 더 써서 큰 시간 아끼는 자리예요."

### ① PRD 사고법 + 본인 앱 컨셉 정의서 (25분, 한 묶음)

PRD 자체는 비전공자도 5~10분이면 1차 작성 가능. 길게 끌지 말고 한 묶음으로 빠르게 끝내고 Lovable 실습 시간 최대 확보.

- 강사 시연 (5분): PRD 생성 프롬프트 던지기 → 결과 함께 읽기
  ```
  나는 [컨셉]을 만들고 싶다.
  비개발자도 이해할 수 있는 1페이지 PRD를 만들어줘.
  - 타겟 사용자 / 해결하려는 문제 / 핵심 기능 3개 / 화면 2~3개 구성
  ```
- 본인 PRD 작성 (15분): 어제 메모 → 1단락 → ChatGPT/Claude에 던지기 → 결과 손질
- AI 자체 검증 (5분): 본인 PRD를 같은 챗봇에 다시 던져 "모호한 부분 / 빠진 정보 / 4일 안에 못 만들 기능" 점검
- 막히는 사람 → 강사가 미리 준비한 **5가지 백업 PRD** 그대로 사용 OK
  - 동네 식당 모음 / 운동 기록 일기 / 가계부 / 독서 메모 / 공구 모임

### ② Lovable 첫 화면 (55분)

- 강사 시연 (10분):
  - Lovable 새 프로젝트 → PRD 통째로 붙여넣기 → 결과 보기
  - 색상 / 폰트 / 레이아웃 한 메시지로 동시에 수정 요청하는 법
  - 막혔을 때 → "이전 화면으로 돌아가기" 사용법
- 수강생 실습 (45분): 본인 PRD로 메인 + 서브 1개 띄우기
- 진도 못 따라오는 사람 → 강사 준비한 백업 PRD 그대로 사용

### ③ Lovable 반복 개선 — 풀 사용자 흐름 (80분, **이 블록이 핵심**)

첫 화면이 떴다면 그건 출발점. 진짜 앱처럼 보이려면 반복 개선이 필요. 80분 풀로 사용.

- **사용자 흐름 그리기** (5분): 종이/메모장에 4~6단계 그림 그리기 (홈 → 입력 → 목록 → 상세 → 수정/삭제)
- **빠진 화면 한 메시지로 묶어 만들기** (20분): 입력 / 상세 / 수정 / 삭제 모달 등 3~4화면을 한 메시지로 추가
- **PRD v2 재투입** (15분): 첫 결과 보고 떠오른 잔소리 5~10개를 PRD에 추가 → 다시 던지기. 이 사이클 1~2회
- **콘텐츠 풍성하게** (15분): 샘플 데이터 5→20개 / Unsplash 자동 이미지 / ChatGPT로 한국어 카피 다듬기
- **인터랙션 상태** (15분): 빈 상태(empty) / 로딩(skeleton·스피너) / 에러(토스트) 중 최소 2개 추가
- **본인 폰에서 미리보기** (5분): Lovable 공유 URL → 폰에서 직접 만져 보기. **수강생에게 가장 짜릿한 순간**
- **풀 사이클 점검** (5분): 처음부터 끝까지 사용자 시점으로 한 번 돌려 보기

### ⑥ (심화·선택) Claude Code 사용법 — 시간 남는 사람용

진도 빠르게 끝난 사람·도구 욕심 있는 사람용 보너스 챕터. 진도 못 따라가는 사람은 건너뛰어도 무방.

- **Claude Code = 본인 컴퓨터에서 동작하는 Claude** (Lovable 졸업 후 다음 단계 도구)
- **하네스 엔지니어링 개념** — 프롬프트 엔지니어링이 "한 번 잘 묻기"라면 하네스 엔지니어링은 "잘 묻는 환경을 미리 잡아두기"
- **CLAUDE.md 작성법** — 프로젝트 규칙서. `/init` 자동 생성 → 본인 손으로 다듬기. 5가지 작성 원칙
- **Skills** — 자주 쓰는 작업을 슬래시 명령어로 등록 (예: `/commit`, `/pr`, `/issue`)
- **Hooks** — 특정 시점 자동 실행 (PreToolUse / PostToolUse / Stop). 파일 저장 시 prettier 자동 등
- **Agents** — 영역별 전문 서브 에이전트 (planner / code-reviewer / security-reviewer 등)
- **MCP** — 외부 도구 연결 표준. 한 줄만 언급
- **라이브 데모 (가능하면 강의 중 직접 시연)** — Lovable 앱에 다크모드 토글 추가. 0~7분 단계별 대본은 docs-site claude-code 챕터에 있음. **수강생 임팩트가 가장 크니 시간 남으면 우선 보여주기**
- **터미널 화면 transcript** — claude 진입 화면, 첫 대화, 파일 분석 예시 — "터미널 무서워요" 반응 풀어주는 자료
- **5분 첫 도전 미션** — 빈 폴더에서 hello.html 만들기. 강의 후 집에서 시도 권장
- **추천 학습 순서** — 1주차: claude로 그냥 대화 → 2주차: CLAUDE.md → 3주차: Skill 1개 → 4주차: Hook 1개 → 5주차+: Agent/MCP

강의 중에는 시간이 남는 만큼 위 항목 일부만 시연. 전체는 docs-site의 [(심화) Claude Code 사용법](/day2/claude-code) 챕터에서 자율 학습.

### 자주 나오는 막힘 + 즉답

- **늦게 도착하시는 분** → 첫 챕터 [아이디어 점검](/day2/idea-check)이 따라잡기 시간을 겸함. 18:25까지 도착하면 무리 없이 합류 가능.
- **너무 큰 아이디어 가져오신 분** → "Red Flag 8가지" 표 보여주고 자르기 3전략 안내. 본인이 못 자르면 백업 PRD 권유.
- **"이거 4일 안에 가능한가요?"** → 헷갈리는 함정 9개 표(공유·알림·결제·검색·추천 등)에서 본인 기능 위치 확인하게 유도.
- **아이디어가 아예 없는 분** → 5가지 발굴 질문(일과 5분 단위 / 가족 불평 / 안 쓰는 앱 / 자주 쓰는 엑셀 / 부모님이 못 쓰시는 앱) 시도. 그래도 없으면 백업 PRD.
- **PRD 단계에서 시간 다 쓰는 사람** → 25분 묶음 안에 강제로 끝내기 ("PRD는 만들면서 계속 고치면 된다"). 못 끝나면 백업 PRD 사용.
- **Lovable 무료 한도 소진** → v0 또는 Bolt 백업으로 즉시 전환 안내. PRD 그대로 재사용.
- **"내 화면이 이상하게 나와요"** → 대부분 PRD가 모호한 게 원인. **AI 자체 검증 프롬프트** (concept.mdx) 한 번 돌려보게 유도.
- **반복 개선 사이클을 한 번도 안 돌리는 사람** → 첫 결과에 만족해서 PRD v2를 안 함. **"화면 보고 떠오른 잔소리 5개"** 적게 시키기.
- **빈/로딩/에러 상태 빠뜨리는 사람** → "데모용 가짜 앱 vs 진짜 앱" 차이 강조. 빈 상태 1개라도 무조건 추가.
- **폰에서 안 보고 모니터에서만 만족하는 사람** → 21:00 즈음 강제로 폰 미리보기 시간 잡기.
- **20명 동시 사용** → Lovable 서버 응답 느려질 수 있음 → 휴식 시간 유연하게.

- **"이대로 앱스토어에 올릴 수 있나요?"** → 웹앱이라 앱스토어 X, 인터넷 주소(URL)로 공유 가능. 모바일 앱은 별도 과정
