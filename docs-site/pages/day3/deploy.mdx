# Lovable로 배포하기

## 이 챕터의 목표

본인 앱을 **인터넷에 띄웁니다**. 진짜 도메인이 생기고, 핸드폰으로도 접속할 수 있게 됩니다.

옛날엔 "GitHub → Vercel → 환경 변수 등록" 흐름이 필요했지만, **이제 Lovable의 Publish 버튼 한 번이면** 끝납니다.

## 작업 흐름

```
Lovable 에디터 우측 상단 "Publish" 버튼 클릭
   ↓
1~2분 기다림
   ↓
xxx.lovable.app 같은 진짜 URL 발급
   ↓
끝. 핸드폰에서도 즉시 접속 가능.
```

<Callout type="info">
**Vercel을 따로 설정할 필요 없음**. Lovable Cloud(백엔드)와 Lovable의 호스팅이 한 묶음이라, Publish 버튼 하나로 프론트엔드 + 백엔드가 동시에 인터넷에 떠 있게 됩니다.
</Callout>

## 단계 1 — Publish 버튼 누르기

Lovable 에디터 화면 **우측 상단**에 **Publish** 버튼이 있습니다.

```
[ Code ] [ Preview ] [ Cloud ] ...                 [ Publish ▼ ]
                                                      ↑ 여기
```

클릭하면 작은 패널이 뜨고 다음을 선택할 수 있습니다.

| 옵션 | 설명 |
| --- | --- |
| **Project URL** | `your-project.lovable.app` 형식의 무료 URL |
| **Visibility** | Public(누구나 접속) / Private(나만) |
| **Custom domain** | 본인이 산 도메인 연결 (선택) |

**Publish** 버튼을 한 번 더 누르면 배포가 시작됩니다.

## 단계 2 — 1~2분 대기

배포 진행 상황이 표시됩니다.

```
빌드 중...
   ↓ 1~2분
배포 성공! ✅
   ↓
URL 발급: https://your-project.lovable.app
```

URL이 나오면 클릭해서 본인 앱이 진짜로 떠 있는지 확인하세요.

<Callout type="info">
**핸드폰을 꺼내서 그 URL을 입력해 보세요.** 본인이 만든 앱이 핸드폰 화면에 떠 있는 모습은 강의에서 가장 인상 깊은 순간입니다. 사진 한 장 찍어 두세요.
</Callout>

## 단계 3 — 배포된 URL에서 회원가입·CRUD 재시도

배포된 URL에서 다음을 한 번 더 시도하세요.

- [ ] 회원가입 (에디터에서 만든 계정과 별개로 새 계정 만들어 보기)
- [ ] 로그인
- [ ] 데이터 1건 입력 → 저장
- [ ] 새로고침 후에도 그대로 보임
- [ ] 로그아웃 → 다시 로그인 → 데이터가 그대로

여기까지 다 되면 **진짜로 인터넷에 살아 있는 본인 앱**이 완성된 겁니다.

## 자동 재배포 — 코드를 바꾸면?

Lovable 에디터에서 채팅으로 화면을 수정하면, **자동으로 다시 배포되지는 않습니다**. 변경 사항을 인터넷에 반영하려면:

1. 에디터에서 수정 → 미리보기에서 결과 확인
2. **Publish 버튼을 다시 한 번** → 새 버전이 배포됨

> 이 점은 옛날 Vercel 흐름(코드 푸시하면 자동 재배포)과 다릅니다. Lovable은 **사용자가 명시적으로 Publish할 때만** 인터넷 버전이 바뀌어요. 미리보기에서 충분히 확인 → Publish, 이 흐름이 안전합니다.

## 커스텀 도메인 (선택)

본인이 산 도메인(예: `myapp.com`)을 연결하고 싶다면:

1. Publish 패널 → **Custom domain** → 도메인 입력
2. Lovable이 알려주는 DNS 레코드(보통 CNAME 또는 A 레코드)를 도메인 등록업체(가비아·Cloudflare·Namecheap 등)에서 추가
3. 1~24시간 안에 본인 도메인으로 접속 가능

자세한 도메인 구매·연결은 [Day 4 — 랜딩페이지·SEO·도메인](/day4/landing-page)에서 다룹니다.

## 흔한 에러 + 해결

### 에러 1: Publish 시도하니 "Free 한도 초과" 메시지

**원인**: Lovable Free 플랜의 빌드/Cloud 한도를 초과

**해결**:

1. 다음 달 한도 리셋 기다리기
2. 또는 Pro 플랜으로 업그레이드

### 에러 2: 배포는 됐는데 빈 화면

**원인 1**: Cloud 연결이 끊김

```text
배포된 버전에서 화면이 비어 있어. Lovable Cloud 연결 상태 확인하고
환경 변수가 빌드에 잘 포함되는지 봐 줘.
```

**원인 2**: 라우팅 문제 (메인 페이지가 로그인 화면으로 무한 리다이렉트 등)

브라우저 개발자도구(F12) → Console 탭에서 빨간 에러 확인 후 Lovable에 알려 주기.

### 에러 3: 회원가입은 되는데 데이터 저장 시 에러

**원인**: RLS 정책이 에디터에선 적용됐는데 배포본에선 빠져 있을 수 있음

**해결**: Cloud 탭 → Database → 해당 테이블 → RLS Policies에서 4개 정책 확인 → 빠진 거 있으면 채팅으로 추가 요청

### 에러 4: "Lovable 배지가 거슬려요"

배포된 앱 우측 하단에 "Edit with Lovable" 작은 배지가 표시됩니다. **Pro 플랜**에서만 끌 수 있습니다 (Free에선 그대로 둬야 함).

## 배포된 URL 메모

배포 성공 후 URL을 메모해 두세요. 이 URL은 다음과 같이 활용됩니다.

- 친구·가족에게 공유 (Day 3 숙제)
- Day 4 발표 자료
- 본인 포트폴리오

```text filename="메모할 값"
배포 URL: https://___________.lovable.app
첫 배포 일시: 2026-04-29 ___:___
```

## 자주 묻는 질문

### Q. "Lovable URL이 마음에 안 들어요"

Publish 패널에서 **subdomain**을 변경할 수 있습니다 (예: `my-fitcheck.lovable.app`). 본인 .com 도메인을 연결하는 건 [Day 4](/day4/landing-page)에서 다룹니다.

### Q. "이 사이트가 다운되면요?"

Lovable의 호스팅은 클라우드 인프라(보통 AWS·Cloudflare 기반) 위에서 돌아가서 안정적입니다. 거의 안 다운됩니다.

### Q. "트래픽이 폭발하면 돈이 청구되나요?"

Free 플랜의 Cloud 한도(월 $25)를 초과하는 트래픽이 들어오면 그때 알림이 옵니다. 일반 데모 앱은 사용자 수백~수천 명 와도 한도 안에 들어와요.

### Q. "사용자가 많아지면 어떻게 봐요?"

Cloud 탭 → **Auth / Users**에서 가입자 수가 보입니다. 또는 Lovable 대시보드 → **Analytics** (이 옵션이 있다면)에서 일일 방문자 추이 확인.

### Q. "Vercel이나 자체 서버에 옮길 수 있어요?"

가능하지만 추가 작업이 필요합니다. [(번외) 직접 Supabase·Vercel 운영하기](/day3/advanced-supabase-vercel)를 참고하세요.
