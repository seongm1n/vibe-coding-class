# Day 3 숙제

## 1. 친구 1명에게 본인 앱 공유 — 회원가입까지

배포된 본인 앱 URL을 친구·가족에게 보내고 **회원가입까지 시켜 보세요**.

### 왜 이걸 하나요?

- 본인이 만든 앱이 **정말 인터넷에 살아 있다**는 걸 체감하는 가장 빠른 방법입니다.
- 다른 사람이 가입하면 Supabase Authentication > Users에 사용자가 늘어나는 것을 직접 봅니다.
- 친구가 막히는 부분이 있다면 그게 진짜 문제입니다 → 다음에 고칠 거리가 생깁니다.

### 카톡으로 보낼 메시지 예시

```text filename="친구에게 보낼 메시지"
[앱 이름] 만들었어. 한 번만 가입해 줄 수 있어?
[Vercel URL]

▷ 5분이면 끝나
▷ 회원가입 → 데이터 한 개만 입력해 보면 돼
▷ 잘 안 되는 부분 있으면 알려 줘 (꼭)
```

## 2. "내 앱이 누구에게 얼마를 받으면 좋을까?" 한 줄 메모

내일(Day 4)은 **수익화**를 다룹니다. 다음 두 가지를 한 줄씩 적어 오세요.

```text filename="수익화 메모 양식"
[1] 누가 돈을 낼 사람인가?
   예: "헬스장 가입자 30명 정도 있는 동네 PT 강사들"

[2] 얼마를 낼만 한가?
   예: "월 5,000원 또는 평생 9,900원 일회성"
```

### 좋은 예시

| 앱 | 누가 돈 낼 사람 | 얼마 |
| --- | --- | --- |
| FitCheck (운동 기록) | 헬스장 사장님 (회원 30명 관리용) | 월 9,900원 (헬스장 단위) |
| 동네 식당 모음 | 광고주(식당 사장) — 사용자 무료 | 식당당 월 19,000원 |
| 가계부 | 프리랜서 (영수증 정리 시간 절약) | 월 3,000원 / 평생 19,000원 |
| 독서 메모 | 독서 모임 운영자 (회원 관리) | 월 5,000원 |
| 공구 모임 | 사용자 (편의 비용) | 거래당 1% 수수료 |

<Callout type="info">
**정답은 없습니다.** 이 메모는 내일 강의에서 함께 다듬을 재료입니다. 첫 직감만 적어 오세요.
</Callout>

## 3. 진도 못 따라온 부분 정리

오늘 어디까지 됐는지 본인 상태를 명확히 적어 두세요. 내일 시작 시 어디부터 도와드릴지 알 수 있습니다.

```text filename="진도 메모 양식"
[√] 회원가입·로그인: 완료 / 진행 중 / 못함
[√] 데이터 저장: 완료 / 진행 중 / 못함
[√] 데이터 불러오기: 완료 / 진행 중 / 못함
[√] 배포 (Vercel URL 발급): 완료 / 진행 중 / 못함
[√] 핸드폰에서 접속: 완료 / 진행 중 / 못함

[ 막힌 곳 ]
- 정확히 어디서 막혔는지 한 줄
- 보였던 에러 메시지가 있으면 그대로 복사

[ 다음에 시도할 것 ]
- 강사·동료가 도와줄 수 있게 본인이 다음에 무엇을 해 보고 싶은지
```

## 4. (선택) Vercel 도메인을 다른 곳에도 공유

본인이 SNS를 한다면 **인스타 스토리·트위터·블로그**에 공유해 보세요.

```text filename="공유 메시지 예시"

[앱 한 줄 설명]

```

이게 의외로 효과적인 마케팅 연습이 됩니다. 본인 앱을 한 줄로 설명할 수 있게 됩니다.

## 오늘 한 일 정리

오늘 배운 것을 한 번 다시 떠올려 보세요.

- [ ] 프론트엔드 / 백엔드 / DB의 차이를 한 문장으로 설명할 수 있다
- [ ] BaaS가 뭐고 왜 비전공자에게 중요한지 안다
- [ ] 환경 변수가 뭐고 왜 필요한지 안다
- [ ] Supabase에 회원가입·로그인을 붙였다
- [ ] 데이터를 저장하고 다시 불러왔다 (CRUD 한 사이클)
- [ ] Vercel로 인터넷에 앱을 띄웠다

이 모든 게 4일 만에 가능하다는 사실, 시작 전엔 상상도 못하셨을 겁니다.
