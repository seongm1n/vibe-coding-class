# Supabase 프로젝트 만들기

## 이 챕터의 목표

Supabase에 새 프로젝트를 만들고, 다음 4가지를 확인합니다.

1. Authentication (인증) 메뉴
2. Table Editor (테이블 편집기)
3. API Keys (API 키 — 환경 변수에 들어갈 값)
4. Project URL (프로젝트 주소)

## 단계 1 — Supabase 가입 (어제 안 하셨으면)

1. [https://supabase.com](https://supabase.com) 접속
2. **"Start your project"** 클릭
3. **"Continue with GitHub"** 선택 → GitHub 로그인
4. 권한 승인

GitHub 계정으로 가입하면 1분 안에 끝납니다.

## 단계 2 — 새 프로젝트 만들기

1. Supabase 대시보드에서 **"New project"** 클릭
2. 다음 정보를 입력합니다.

| 항목 | 입력값 | 설명 |
| --- | --- | --- |
| **Organization** | (자동 생성된 것 그대로) | 그대로 두기 |
| **Project name** | 예: `fitcheck`, `my-app` | 영어 소문자 + 하이픈, 짧게 |
| **Database Password** | 자동 생성 버튼 클릭 추천 | **메모해 두기** (아래 주의 참고) |
| **Region** | **Northeast Asia (Seoul)** | 한국에서 가장 빠름 |
| **Pricing Plan** | **Free** | 무료 플랜 |

3. **"Create new project"** 클릭 → 1~2분 대기 (Supabase가 백엔드를 생성합니다)

<Callout type="warning">
**Database Password를 꼭 메모해 두세요.** 자주 쓰지는 않지만 나중에 DB에 직접 접근할 일이 생기면 필요합니다. 잃어버려도 다시 설정할 수 있긴 합니다.
</Callout>

<Callout type="info">
**왜 Seoul 리전인가요?** 서버가 한국에 있으면 한국 사용자에게 빠릅니다. 서버가 미국에 있으면 사용자→미국 왕복으로 응답 속도가 느려집니다 (200~300ms vs 20~50ms).
</Callout>

## 단계 3 — 4가지 핵심 메뉴 확인

프로젝트가 생성되면 왼쪽 사이드바에 다음 메뉴가 보입니다. 하나씩 클릭해 보세요.

### 1. Authentication (사람 모양 아이콘)

회원가입한 사용자 목록이 보이는 곳입니다. 지금은 비어 있을 겁니다.

```
Authentication > Users
- 가입한 사용자가 보이는 표
- 이메일 / 가입일 / 마지막 로그인 표시
```

### 2. Table Editor (테이블 모양 아이콘)

데이터를 표(엑셀)처럼 관리하는 곳입니다. 지금은 비어 있습니다. 다음 챕터에서 첫 테이블을 만들 겁니다.

```
Table Editor
- 테이블 목록 (지금은 비어 있음)
- "New table" 버튼으로 만들기
```

### 3. SQL Editor (DB 메뉴 안)

코드로 DB를 조작할 수 있는 곳입니다. 우리는 **거의 안 씁니다.** Lovable이 알아서 처리해 줍니다.

### 4. Project Settings → API

가장 중요한 곳입니다.

1. 왼쪽 사이드바 맨 아래 **⚙️ Settings** → **API**
2. 다음 두 값을 찾으세요.

```
[ Project URL ]
https://xxxxxxxxxxxxx.supabase.co
↑ 여기 본인 프로젝트의 고유 URL

[ Project API keys ]
- anon public: eyJxxxx... (긴 문자열)
- service_role: eyJxxxx... (긴 문자열, 절대 공개 금지)
```

## 단계 4 — API 키 메모 (Lovable에 곧 넣을 값)

다음 두 값을 본인 메모장에 복사해 두세요. 이걸 잠시 후 Lovable 환경 변수에 넣을 겁니다.

```text filename="메모할 값"
SUPABASE_URL = https://xxxxxxx.supabase.co
SUPABASE_ANON_KEY = eyJxxxxxxxxxxxxxxx...

(service_role 키는 무시하세요)
```

<Callout type="error">
**절대 SNS·블로그·GitHub에 올리지 마세요.**
- ANON 키는 비교적 덜 위험하지만 그래도 노출 안 하는 게 원칙입니다.
- SERVICE_ROLE 키는 노출되면 누구나 본인 DB를 마음대로 쓸 수 있습니다. **공개 절대 금지.**
</Callout>

## 자주 하는 실수

<Callout type="error">
**실수 1: URL 끝에 슬래시(/)를 붙임**
`https://xxxxxxx.supabase.co/` ← **잘못**
`https://xxxxxxx.supabase.co` ← **맞음**
끝 슬래시 하나 때문에 연결이 안 되는 경우가 많습니다.
</Callout>

<Callout type="error">
**실수 2: ANON 키와 SERVICE_ROLE 키를 헷갈림**
**ANON 키**를 쓰세요. SERVICE_ROLE 키는 절대 화면에서 쓰지 마세요.
</Callout>

<Callout type="error">
**실수 3: 키 복사 시 마지막 글자 빠뜨림**
키가 매우 길어서 복사할 때 끝부분이 잘리기 쉽습니다. **"Copy" 버튼**을 사용하세요. 손으로 드래그하지 마세요.
</Callout>

## 체크포인트

## 자주 묻는 질문

### Q. "프로젝트 만드는 데 시간이 너무 걸려요"

보통 1~2분이지만, 가끔 3~5분 걸립니다. 그동안 다른 작업(예: 메모 정리, Lovable 화면 확인)을 하세요.

### Q. "프로젝트 이름을 바꿀 수 있어요?"

**Project Settings > General**에서 이름은 바꿀 수 있지만, **Project URL은 변경 불가**입니다. 이름이 살짝 어색해도 그대로 쓰는 게 편합니다.

### Q. "여러 프로젝트를 만들 수 있어요?"

무료 플랜에서 **2개**까지 만들 수 있습니다. 강의에서는 1개로 충분합니다. 너무 많이 만들면 헷갈리니 1개로 끝까지 가세요.

### Q. "이 키들이 노출되면 어떻게 해요?"

**Project Settings > API**에서 키를 다시 생성(Regenerate)할 수 있습니다. 새 키가 발급되면 옛 키는 무효화됩니다. 단, Lovable·Vercel 환경 변수도 새 값으로 바꿔야 합니다.
