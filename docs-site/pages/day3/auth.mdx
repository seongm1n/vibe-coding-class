# 회원가입·로그인 붙이기

## 이 챕터의 목표

본인 앱에 **회원가입과 로그인**을 붙입니다. 사용자가 이메일·비밀번호로 가입하고, 다시 로그인할 수 있게 만듭니다.

이 작업이 끝나면 Supabase Authentication 메뉴에 사용자가 1명(=본인)이 등록되어 보입니다.

## 작업 흐름

```
1. Lovable 환경 변수에 Supabase URL/KEY 입력
   ↓
2. Lovable에 "Supabase로 회원가입/로그인 추가해 줘" 시키기
   ↓
3. 미리보기에서 회원가입 → 로그인 시도
   ↓
4. Supabase Authentication에서 본인 사용자 확인
```

## 단계 1 — Lovable에 환경 변수 입력

Lovable 프로젝트에는 **환경 변수 관리 화면**이 있습니다. 위치는 다음과 같습니다.

```
프로젝트 화면 우측 상단 ⚙️ 설정 → "Environment Variables" 또는 "Secrets"
```

(UI는 Lovable 업데이트에 따라 살짝 다를 수 있습니다. 안 보이면 강사·보조 인력에게 물어보세요.)

다음 두 값을 추가합니다.

| 이름 | 값 |
| --- | --- |
| `VITE_SUPABASE_URL` 또는 `NEXT_PUBLIC_SUPABASE_URL` | (앞 챕터에서 메모한) Project URL |
| `VITE_SUPABASE_ANON_KEY` 또는 `NEXT_PUBLIC_SUPABASE_ANON_KEY` | (앞 챕터에서 메모한) anon 키 |

<Callout type="info">
**환경 변수 이름은 Lovable이 자동으로 알려 줍니다.** 다음 단계의 프롬프트를 던지면 Lovable이 "이 이름으로 환경 변수를 만들어 주세요"라고 안내해 줄 겁니다. 그 안내대로 따라가는 게 가장 정확합니다.
</Callout>

## 단계 2 — Lovable에 회원가입/로그인 시키기

채팅창에 다음 프롬프트를 붙여 넣습니다.

### 좋은 프롬프트 — 한 번에 모두 시키기

```text filename="✅ 회원가입/로그인 추가 프롬프트"
이 앱에 Supabase로 회원가입/로그인 기능을 추가해 줘.

요구사항:
- 가입 방식: 이메일 + 비밀번호
- 로그인되지 않은 사용자는 메인 화면 대신 로그인 화면이 보이게
- 로그인 후 메인 화면으로 자동 이동
- 메인 화면 우측 상단에 "로그아웃" 버튼

화면 구성:
- 로그인 화면
  - 앱 이름과 로고
  - 이메일 입력
  - 비밀번호 입력 (8자 이상)
  - "로그인" 버튼 (큰 강조 버튼)
  - 아래에 "처음이세요? 회원가입" 링크

- 회원가입 화면
  - 이메일 입력
  - 비밀번호 입력 (8자 이상)
  - 비밀번호 확인 입력
  - "회원가입" 버튼
  - 아래에 "이미 계정 있으세요? 로그인" 링크

기술 요구사항:
- Supabase URL과 ANON KEY는 환경 변수로 관리
- 환경 변수가 없으면 화면에 친절한 안내 메시지 표시
- 가입 후 이메일 인증을 요구하지 않게 (강의 데모용, Supabase 설정에서 끔)

에러 처리:
- 이메일 형식이 틀리면 "이메일 형식이 아닙니다"
- 비밀번호가 짧으면 "비밀번호는 8자 이상이어야 합니다"
- 이미 가입된 이메일이면 "이미 가입된 이메일입니다. 로그인하세요"
- 비밀번호가 틀리면 "이메일 또는 비밀번호가 잘못되었습니다"

언어: 한국어
```

이 프롬프트 한 방이면 회원가입·로그인이 다 들어갑니다.

### 다 같이 — Supabase에서 이메일 인증 끄기

Supabase는 기본적으로 가입 시 **이메일 인증 메일**을 보냅니다. 강의 시간엔 이걸 꺼야 빠르게 진행됩니다. 본인 Supabase 대시보드에서 다 같이 따라 하세요.

1. Supabase 대시보드 → **Authentication** → **Providers**
2. **Email** 클릭
3. **"Confirm email"** 토글을 **OFF**로
4. **Save** 클릭

<Callout type="warning">
**실제 서비스 운영 시에는 이메일 확인을 켜는 게 안전합니다.** 가짜 이메일로 가입하는 것을 막아 주기 때문입니다. 강의 데모 단계에서만 꺼 둡니다.
</Callout>

## 단계 3 — 회원가입 + 로그인 시도

미리보기에서 직접 가입해 봅니다.

1. 미리보기 화면에서 **"회원가입"** 클릭
2. 본인 이메일 + 비밀번호 (8자 이상) 입력
3. **"회원가입"** 버튼 클릭
4. 자동으로 메인 화면으로 이동되는지 확인

### 잘 됐는지 확인하기

Supabase 대시보드 → **Authentication > Users**

```
이메일 | 가입일 | 마지막 로그인
your@email.com | 2026-04-29 ... | 방금
```

여기에 본인 이메일이 보이면 성공입니다.

## 자주 발생하는 에러와 해결

### 에러 1: "Invalid API key"

**원인**: 키 복사 시 끝부분이 잘렸거나, URL/키가 서로 바뀌어 들어감

**해결**:
1. Supabase Project Settings > API에서 다시 복사
2. **Copy 버튼** 사용 (드래그하지 말 것)
3. 환경 변수 값 다시 입력

### 에러 2: "Failed to fetch" 또는 네트워크 에러

**원인**: URL 끝에 슬래시(/)가 붙어 있음

**해결**: URL 마지막 슬래시 제거
```
잘못: https://xxxxx.supabase.co/
맞음: https://xxxxx.supabase.co
```

### 에러 3: "Email not confirmed"

**원인**: Supabase 이메일 인증이 켜져 있는데 인증 메일을 안 누름

**해결 1**: 이메일 받은편지함에서 인증 메일 클릭
**해결 2**: Authentication > Providers > Email에서 "Confirm email" OFF (강의 데모용)

### 에러 4: 환경 변수가 안 먹음

**원인**: Lovable이 환경 변수 이름을 잘못 인식

**해결**:
1. Lovable이 콘솔/에러창에서 "어떤 이름의 환경 변수를 찾는지" 메시지를 보여 줍니다
2. 그 이름과 동일하게 환경 변수를 등록
3. 페이지 새로고침

### 에러 5: 로그인은 되는데 메인 화면으로 이동 안 됨

**원인**: 라우팅(화면 이동) 처리가 안 됨

**해결 프롬프트**:
```text
로그인 성공 후 메인 화면(/)으로 자동 이동되게 해 줘.
또한 로그인 안 된 상태에서 메인 화면에 접근하면 자동으로 로그인 화면(/login)으로 보내 줘.
```

## 추가 — 소셜 로그인 (선택, 시간 남으면)

이메일 가입 외에 **Google 로그인**도 쉽게 추가할 수 있습니다.

```text filename="Google 로그인 추가 프롬프트"
로그인 화면 아래에 "Google로 로그인" 버튼을 추가해 줘.
Supabase OAuth로 처리.
```

다만 Google OAuth는 Supabase에서 별도 설정이 필요합니다 (Google Cloud Console에서 클라이언트 ID 발급). 강의 시간엔 생략하고 **이메일 로그인만으로 충분**합니다.

## 자주 묻는 질문

### Q. "비밀번호 잊어버리면 어떻게 해요?"

Supabase는 **비밀번호 재설정** 기능도 기본 제공합니다. 다음 프롬프트로 추가할 수 있습니다.

```text
로그인 화면에 "비밀번호 찾기" 링크 추가.
이메일을 입력하면 Supabase가 재설정 이메일을 보내 주는 흐름.
```

(시간 남을 때 시도)

### Q. "다른 사람 비밀번호를 볼 수 있어요?"

**아니요. 절대로 못 봅니다.** Supabase는 비밀번호를 **해시(hash)** 라는 방식으로 저장합니다. 본인 비밀번호도 본인이 다시 못 봅니다(잊어버리면 재설정해야 함).

### Q. "회원가입한 사용자를 삭제할 수 있어요?"

Supabase 대시보드 → Authentication > Users에서 사용자를 클릭하면 **Delete user** 버튼이 있습니다. 강의 데모 후 정리할 때 유용합니다.
