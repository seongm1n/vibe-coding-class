# 회원가입·로그인 붙이기

## 이 챕터의 목표

본인 앱에 **회원가입과 로그인**을 붙입니다. 사용자가 이메일·비밀번호로 가입하고, 다시 로그인할 수 있게 만듭니다.

이 작업이 끝나면 Lovable의 **Cloud 탭 → Auth / Users**에 사용자가 1명(=본인)이 등록되어 보입니다.

## 작업 흐름

```
1. 채팅에 한 번에 "회원가입/로그인 붙여 줘" 프롬프트 던지기
   ↓
2. Lovable이 화면 + 백엔드 처리 코드 + 라우팅까지 자동 작성
   ↓
3. 미리보기에서 회원가입 → 로그인 시도
   ↓
4. Cloud 탭 → Auth / Users 에서 본인 사용자 확인
```

<Callout type="info">
**환경 변수 설정 단계 없음**. 앞 챕터에서 Lovable Cloud를 연결했다면, 인증에 필요한 키는 이미 자동 주입되어 있습니다. 채팅 한 번이면 끝납니다.
</Callout>

## 단계 1 — 한 번에 모두 시키기

채팅창에 다음 프롬프트를 그대로 붙여 넣습니다.

```text filename="회원가입/로그인 추가 프롬프트"
이 앱에 회원가입/로그인 기능을 추가해 줘. (Lovable Cloud의 인증 사용)

요구사항:
- 가입 방식: 이메일 + 비밀번호
- 로그인되지 않은 사용자는 메인 화면 대신 로그인 화면이 보이게
- 로그인 후 메인 화면으로 자동 이동
- 메인 화면 우측 상단에 로그인된 이메일 + "로그아웃" 버튼

화면 구성:
- 로그인 화면
  - 앱 이름과 로고
  - 이메일 입력
  - 비밀번호 입력 (8자 이상)
  - "로그인" 버튼 (큰 강조 버튼)
  - 아래에 "처음이세요? 회원가입" 링크

- 회원가입 화면
  - 이메일 입력
  - 비밀번호 입력 (8자 이상)
  - 비밀번호 확인 입력
  - "회원가입" 버튼
  - 아래에 "이미 계정 있으세요? 로그인" 링크

기술 요구사항:
- 회원가입 직후 자동 로그인 (이메일 인증 없이 바로 진입, 강의 데모용)
- 세션은 새로고침해도 유지

에러 처리:
- 이메일 형식이 틀리면 "이메일 형식이 아닙니다"
- 비밀번호가 짧으면 "비밀번호는 8자 이상이어야 합니다"
- 이미 가입된 이메일이면 "이미 가입된 이메일입니다. 로그인하세요"
- 비밀번호가 틀리면 "이메일 또는 비밀번호가 잘못되었습니다"

언어: 한국어
```

이 프롬프트 한 방이면 회원가입·로그인이 다 들어갑니다. **메시지 1번으로 끝납니다.**

<Callout type="warning">
**메시지 한도 절약 팁**: Lovable Free 플랜은 일/월 메시지 한도가 있습니다. 위 프롬프트처럼 **한 번에 모든 요구사항을 명시**해서 쪼개지 않는 게 한도 아끼는 핵심입니다.
</Callout>

## 단계 2 — 이메일 인증 끄기 (Lovable Cloud 설정)

Lovable Cloud는 기본적으로 가입 시 이메일 인증을 요구할 수 있습니다. 강의 시간엔 이걸 꺼야 빠르게 진행됩니다.

### 방법 A — 채팅으로 끄기 (가장 빠름)

```text filename="이메일 인증 비활성화 프롬프트"
회원가입 후 이메일 인증 없이 바로 로그인되게 해 줘.
Lovable Cloud의 Auth 설정에서 "Confirm email" 옵션을 꺼 줘.
```

### 방법 B — Cloud 탭에서 직접 끄기

1. **Cloud 탭** → **Auth** 또는 **Authentication** 메뉴
2. **Settings** 또는 **Providers** → **Email**
3. **Confirm email** 토글을 **OFF**
4. 저장

<Callout type="warning">
**실제 서비스 운영 시에는 이메일 인증을 켜는 게 안전합니다.** 가짜 이메일로 가입하는 것을 막아 주기 때문입니다. 강의 데모 단계에서만 꺼 둡니다.
</Callout>

## 단계 3 — 회원가입 + 로그인 시도

미리보기에서 직접 가입해 봅니다.

1. 미리보기 화면에서 **"회원가입"** 클릭
2. 본인 이메일 + 비밀번호 (8자 이상) 입력
3. **"회원가입"** 버튼 클릭
4. 자동으로 메인 화면으로 이동되는지 확인
5. **로그아웃** → 다시 **로그인** 시도

### 잘 됐는지 확인하기 — Cloud 탭에서

1. Lovable 에디터 상단 → **Cloud 탭**으로 이동
2. 좌측 **Auth** 또는 **Users** 메뉴
3. 방금 가입한 이메일이 한 줄로 떠 있어야 함

```
Email                  | Provider | Created at        | Last sign in
your@email.com         | email    | 2026-04-29 14:23  | 방금
```

여기에 본인 이메일이 보이면 **진짜 백엔드와 연결된 것**입니다.

## 자주 발생하는 에러와 해결

### 에러 1: "회원가입은 됐는데 Cloud 탭 Users에 안 떠요"

**원인 1** — Cloud 연결이 제대로 안 됨. 앞 챕터([Lovable Cloud로 백엔드 켜기](/day3/lovable-cloud))로 돌아가 `src/integrations/supabase/client.ts`가 있는지 확인.

**원인 2** — Lovable Cloud가 아닌 외부(개인) Supabase에 연결된 경우. 채팅에 다음을 던져 확인:

```text
지금 인증이 Lovable Cloud로 연결돼 있어, 아니면 다른 Supabase로 연결돼 있어?
src/integrations/supabase/client.ts의 URL 보여 줘.
```

### 에러 2: "Email not confirmed"

**원인**: 이메일 인증이 켜져 있는데 인증 메일을 안 누름

**해결 1**: 단계 2의 "이메일 인증 끄기" 다시 시도
**해결 2**: 받은편지함에서 인증 메일 클릭

### 에러 3: 로그인은 되는데 메인 화면으로 이동 안 됨

**원인**: 라우팅(화면 이동) 처리 누락

**해결 프롬프트**:

```text
로그인 성공 후 메인 화면(/)으로 자동 이동되게 해 줘.
또한 로그인 안 된 상태에서 메인 화면에 접근하면 자동으로 로그인 화면(/login)으로 보내 줘.
세션 상태가 바뀌면 즉시 라우팅도 갱신되게.
```

### 에러 4: "Invalid API key" 또는 "Failed to fetch"

**원인**: Cloud 연결이 끊겼거나 환경 변수가 비어 있음

**해결**:

1. Code 탭 → `.env` 파일 열어 보기
2. `VITE_SUPABASE_URL` / `VITE_SUPABASE_PUBLISHABLE_KEY` 두 줄이 있어야 함
3. 비어 있으면 채팅에 **"Lovable Cloud 다시 연결해 줘"** 한 번 더

## 추가 — 소셜 로그인 (선택, 시간 남으면)

이메일 가입 외에 **Google 로그인**도 추가 가능.

```text filename="Google 로그인 추가 프롬프트"
로그인 화면 아래에 "Google로 로그인" 버튼을 추가해 줘.
Lovable Cloud의 Google OAuth로 처리.
```

다만 Google OAuth는 별도 설정이 필요합니다 (Google Cloud Console에서 클라이언트 ID 발급). 강의 시간엔 생략하고 **이메일 로그인만으로 충분**합니다.

## 자주 묻는 질문

### Q. "비밀번호 잊어버리면 어떻게 해요?"

**비밀번호 재설정** 기능도 기본 제공됩니다. 다음 프롬프트로 추가할 수 있습니다.

```text
로그인 화면에 "비밀번호 찾기" 링크 추가.
이메일을 입력하면 Lovable Cloud가 재설정 이메일을 보내 주는 흐름.
```

(시간 남을 때 시도)

### Q. "다른 사람 비밀번호를 볼 수 있어요?"

**아니요. 절대로 못 봅니다.** 비밀번호는 **해시(hash)** 방식으로 저장됩니다. 본인 비밀번호도 본인이 다시 못 봅니다(잊어버리면 재설정해야 함).

### Q. "회원가입한 사용자를 삭제할 수 있어요?"

Cloud 탭 → **Auth / Users**에서 사용자 행을 클릭하면 **Delete** 메뉴가 있습니다. 강의 데모 후 정리할 때 유용.

### Q. "Cloud Free 플랜에서 회원이 몇 명까지 가입돼도 무료예요?"

워크스페이스당 월 $25 어치 Cloud 사용량이 제공됩니다. 인증·DB·Storage 합쳐서 그 안에 들면 무료. 강의 데모 + 친구 1~2명 가입 수준이면 거의 안 깎입니다.
