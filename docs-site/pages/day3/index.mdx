## 오늘 챙겨 갈 핵심 개념

### 프론트엔드 / 백엔드 / DB

- **프론트엔드** = 사용자가 보는 화면 (지금까지 Lovable로 만든 것)
- **백엔드** = 화면 뒤에서 데이터를 처리하는 곳 (오늘 Lovable Cloud가 그 역할)
- **데이터베이스(DB)** = 데이터가 실제로 쌓이는 창고
- **비유**: 식당 = 손님이 보는 홀(프론트) + 주방(백엔드) + 냉장고(DB)

### BaaS = 백엔드를 빌려 쓰기

옛날엔 서버를 직접 깔고 DB를 직접 운영해야 했습니다. 비전공자에겐 거의 불가능했죠. 지금은 **BaaS(Backend as a Service)** 가 회원가입·DB·파일 저장을 다 해 줍니다.

> **이게 비전공자가 4일 만에 앱을 만들 수 있게 된 결정적 이유입니다.**

이 강의에선 **Lovable Cloud**(엔진은 Supabase 기반)를 사용합니다. Lovable 안에서 **Cloud 탭** 한 번 켜면 백엔드 + DB가 자동으로 준비됩니다.

### 환경 변수 (env)

API 키 같은 비밀값은 **코드에 직접 적지 않고** 따로 관리합니다. Lovable Cloud는 이 키를 자동으로 주입해 주므로 본인이 손으로 입력할 일은 거의 없어요. 다만 **개념은 알아 두기** — 노출되면 누군가 내 계정으로 요금을 쓸 수 있어요. **비밀번호 같은 것**입니다.

### 배포 = 인터넷에 띄우기

지금까지는 Lovable 미리보기에서만 동작합니다. **Publish** 버튼 한 번 누르면 진짜 도메인이 생기고 누구나 접속할 수 있습니다. **"내 노트북을 꺼도 살아 있는 앱"** 이 됩니다.
