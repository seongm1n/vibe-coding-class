# 이미지 생성

## 이 챕터의 목표

AI 이미지 생성 도구 3개를 체험하고, **본인 앱 로고 1개**를 만들어 봅니다. 이 로고는 Day 2에 만들 화면에 그대로 사용합니다.

## 이미지 생성 도구는 왜 여러 개를 쓰나요?

도구마다 잘 만드는 그림 스타일이 다릅니다. 같은 프롬프트도 결과가 완전히 다릅니다.

| 도구 | 장점 | 약점 | 잘 맞는 용도 |
| --- | --- | --- | --- |
| [Nano Banana (Gemini)](https://gemini.google.com) | 무료, 한국어 프롬프트가 잘 먹음, 빠름 | 사실적인 사진은 약함 | 일러스트, 컨셉 이미지 |
| [Bing Image Creator](https://www.bing.com/images/create) | DALL-E 3 무료 사용, MS 계정만 있으면 됨 | 매일 한도 있음 | 사실적인 그림, 스타일 다양 |
| [Ideogram](https://ideogram.ai) | **이미지 안의 글자**를 잘 그림 | 무료 한도 빠르게 소진 | 로고, 포스터, 글자 들어간 그림 |

<Callout type="info">
**Ideogram은 로고 만들기에 특히 좋습니다.** 다른 도구는 이미지 안에 한글·영어 글자를 넣으면 글자가 깨져서 나오는 경우가 많습니다.
</Callout>

## 다 같이 따라 하기

### 사전 준비

브라우저에 **새 탭 3개**를 엽니다. 각 탭에 다음을 띄우세요.

| 탭 | 사이트 | URL | 가입 방법 |
| --- | --- | --- | --- |
| 1 | Nano Banana (Gemini) | https://gemini.google.com | Google 계정 로그인 |
| 2 | Bing Image Creator | https://www.bing.com/images/create | Microsoft 계정 로그인 |
| 3 | Ideogram | https://ideogram.ai | Google 또는 이메일 가입 |

3개 모두 로그인된 상태가 되면 OK. **가입 안 된 탭이 있으면 일단 두고 다음 단계로 갑니다.** 시간 안에 따라가는 게 중요합니다.

### 1단계 — 한국어 의도를 영어 프롬프트로 변환

이미지 도구는 **영어 프롬프트가 잘 먹습니다.** 한국어로 적은 의도를 ChatGPT에 영어 프롬프트로 변환시켜 보겠습니다.

**ChatGPT** (또는 Claude) 탭에 다음을 붙여 넣으세요.

```text filename="영어 이미지 프롬프트로 변환 — 라이브"
다음 한국어 설명을 이미지 생성용 영어 프롬프트로 바꿔줘.
스타일·색상·구성·배경·분위기까지 빠짐없이 포함해서.
한국어로는 답하지 말고 영어 프롬프트만 줘.

[한국어 설명]
점심 시간 식당 추천 앱의 로고를 만들고 싶다.
- 미니멀, 플랫 디자인, 둥근 모서리
- 짙은 주황(#c8401a)과 흰색 위주
- 식당 / 점심 메뉴를 떠올릴 수 있는 심볼 (그릇, 젓가락, 시계 등 1개 결합)
- 정사각형 구도, 모바일 앱 아이콘으로 쓸 수 있게
- 배경은 단색 또는 투명
- 글자가 들어가도 OK ("Lunch" 정도)
```

ChatGPT가 만들어 준 영어 프롬프트를 **메모장에 복사** 해 두세요. 이걸 잠시 후 3개 도구에 똑같이 던질 겁니다.

<Callout type="info">
**본인 앱 컨셉이 이미 있으면 그걸로** 위 한국어 설명만 본인 컨셉으로 바꿔서 다시 시키세요. 없으면 위의 "점심 식당 추천" 컨셉으로 그대로 따라가도 됩니다.
</Callout>

### 2단계 — Nano Banana (Gemini)에서 생성

#### 진입 흐름

1. **1번 탭(Gemini)** 으로 이동
2. 화면 하단 채팅창 옆에 **"+" 또는 이미지 아이콘** 이 있을 수 있습니다 (UI는 가끔 바뀝니다)
3. 단순히 채팅창에 **방금 메모한 영어 프롬프트** 를 붙여 넣고 **"Create an image: "** 를 앞에 붙입니다.

```text filename="Gemini 입력창에 그대로"
Create an image: [방금 ChatGPT에서 만든 영어 프롬프트 붙여넣기]
```

4. Enter → 1~2분 기다리면 보통 **이미지 4장 정도** 가 나옵니다.

#### 결과 확인

- 4장 중 가장 마음에 드는 1장을 정합니다.
- 마우스 우클릭 → **"이미지 저장"** 으로 다운로드합니다.
- 파일 이름은 알아보기 쉽게 (`logo-nano-banana.png` 같이)

#### 잘 안 됐을 때

- 결과가 너무 어색하면 같은 프롬프트로 **"Generate variations"** 또는 다시 한 번 같은 프롬프트 → 다른 결과가 나옵니다.
- 글자가 깨졌으면 일단 **글자 없이** 만들어 보세요. 글자는 잠시 후 Ideogram에서 시도합니다.

```text filename="Nano Banana 재시도 프롬프트"
다시 만들어 줘. 이번엔 글자는 빼고, 심볼만 더 단순하게.
배경은 흰색 정사각형으로.
```

### 3단계 — Bing Image Creator에서 같은 프롬프트

#### 진입 흐름

1. **2번 탭(Bing Image Creator)** 으로 이동
2. 가운데 큰 입력창에 **방금 ChatGPT에서 만든 같은 영어 프롬프트** 를 붙여 넣습니다.
3. **"Create"** 버튼 클릭 → 30초~1분 대기

#### 결과 확인

- Bing은 보통 **4장**을 한 번에 보여 줍니다.
- DALL-E 3 기반이라 **사실적인 그림**이 Nano Banana보다 강합니다.
- 마음에 드는 1장 클릭 → 우측 상단 다운로드 아이콘.

#### Nano Banana와 비교 포인트

본인 결과를 직접 비교하면서 다음을 체크하세요.

- 어느 쪽이 더 **컬러 밸런스**가 좋은가?
- 어느 쪽이 더 **모던한 느낌**인가?
- 글자가 들어갔다면 어느 쪽이 글자가 안 깨졌나?

### 4단계 — Ideogram에서 같은 프롬프트

#### 진입 흐름

1. **3번 탭(Ideogram)** 으로 이동
2. 가운데 입력창에 **같은 영어 프롬프트** 붙여 넣기
3. (선택) 화면 옆에 **"Aspect ratio"** 1:1, **"Style"** Auto 또는 Design 선택
4. **"Generate"** 클릭 → 30초~1분 대기

#### 결과 확인

- Ideogram이 가장 잘하는 건 **글자가 깨지지 않는 이미지**입니다.
- 만약 다른 두 도구에서 글자(예: "Lunch")가 깨졌다면, **여기서만 글자가 살아 있을** 가능성이 큽니다.
- 마음에 드는 1장 다운로드.

### 5단계 — 3장 비교 + 1장 확정

3개 도구에서 받은 이미지 3장을 한 화면에 띄워 놓으세요.

| 도구 | 강점 | 약점 |
| --- | --- | --- |
| Nano Banana | 한국어 의도 자연스러움, 빠름 | 사실감 약함 |
| Bing | 사실적인 그림, 모던 | 매일 한도, MS 계정 필요 |
| Ideogram | 글자 안 깨짐, 로고에 강함 | 무료 한도 빨리 소진 |

이 중 **본인 앱 로고로 쓸 1장**을 골라 `my-app-logo.png` 라는 이름으로 저장하세요. **Day 2에서 Lovable에 올릴 파일** 입니다.

### 결과 정리

체크포인트:

- [ ] 본인 컴퓨터에 `my-app-logo.png` (또는 비슷한 파일명)이 저장됨
- [ ] 3개 도구 중 어느 게 본인에게 잘 맞는지 감 잡음
- [ ] 어느 도구로 다시 만들지 본인 머릿속에 정해 둠

## 실습 (선택) — 같은 프롬프트, 3개 도구 비교

위 라이브가 끝나고 시간이 남으면, **본인 앱 컨셉을 더 다양한 스타일** 로 시도해 보세요. 아래는 참고용 3가지 프롬프트 버전입니다.

### 프롬프트 예시 — 3가지 버전

#### 나쁜 예

```text filename="나쁜 예"
운동 앱 로고
```

너무 짧습니다. AI가 어떤 스타일로 그릴지, 어떤 색을 쓸지, 어떤 분위기를 낼지 단서가 전혀 없습니다.

#### 보통 예

```text filename="보통 예"
헬스장 출석 체크 앱 로고. 미니멀하고 모던한 스타일.
```

스타일은 명시했지만, 색상·구성·구체적인 요소가 부족합니다. 결과는 평범합니다.

#### 좋은 예

```text filename="좋은 예"
헬스장 회원 출석 체크 앱의 로고를 만들어줘.
- 스타일: 미니멀, 플랫 디자인, 둥근 모서리
- 색상: 짙은 주황(#c8401a)과 흰색 위주, 강조색 1개
- 구성: 체크 마크와 덤벨이 결합된 심볼 + 앱 이름 "FitCheck"
- 배경: 단색 또는 투명
- 용도: 모바일 앱 아이콘으로도 쓸 수 있게 정사각형 구도
```

스타일·색·구성·용도까지 모두 명시했습니다. 결과가 일관되게 나옵니다.

### 예상 결과 비교

| 프롬프트 | 결과 | 부족한 점 |
| --- | --- | --- |
| 나쁜 예 | 스톡 이미지처럼 보이는, 어디서 본 것 같은 평범한 로고 | 본인 앱과 무관함 |
| 보통 예 | 비슷비슷한 로고가 4개 정도 나옴 | 색·이름이 본인 앱과 안 맞음 |
| 좋은 예 | 색·이름·구성이 의도대로 반영된 로고 | 글자가 깨져 나오면 Ideogram 사용 |

### 개선 방법

결과가 마음에 안 들면 다음을 시도해 보세요.

1. **참고 이미지 묘사를 추가** ("토스(Toss) 앱 아이콘 같은 느낌")
2. **부정 표현 사용** ("사진 같은 사실적인 스타일은 피하고")
3. **스타일 키워드 보강** (flat, minimal, vector, geometric, retro 등 영어 스타일 단어가 잘 먹힙니다)
4. **글자 깨지면 Ideogram으로** 다시 시도

```text filename="개선 프롬프트 예시"
앞서 만든 로고에서 다음만 바꿔줘:
- 덤벨 대신 운동화 실루엣
- "FitCheck" 글자는 더 굵고 진하게
- 배경은 투명하게
```

## 실습 2 — 본인 앱 로고 만들기

오늘이 끝나면 본인 앱 아이디어가 아직 정해지지 않았어도 괜찮습니다. **임시 컨셉으로** 일단 1개 만들어 두세요.

### 단계

1. 컨셉 1줄 정하기 (예: "동네 헬스장 운영시간 모음")
2. 위의 "좋은 예" 프롬프트 형식에 본인 컨셉을 채워 넣기
3. 3개 도구 중 가장 결과가 마음에 드는 도구로 1개 확정
4. 이미지를 본인 컴퓨터에 저장 (PNG 형식 권장)
5. 파일명을 알아보기 쉽게 (예: `my-app-logo.png`)

### 체크포인트

- [ ] 본인 컴퓨터에 로고 PNG 파일 1개 저장됨
- [ ] 파일명을 본인이 기억하기 쉽게 정함
- [ ] Day 2에 이 파일을 Lovable에 올릴 예정

<Callout type="warning">
**실수 1: 글자가 깨져서 나옴**
한글이나 영어 글자를 이미지 안에 넣으면 의미 없는 모양으로 나오기 쉽습니다. → **Ideogram**을 쓰거나, 글자 없이 이미지만 만든 뒤 나중에 Lovable이 글자를 따로 넣게 하세요.

<Callout type="warning">

</Callout>

<Callout type="warning">
**실수 3: 무료 한도를 쓸데없는 시도로 다 쓴다**
하루 한도가 정해져 있는 도구가 많습니다. → **프롬프트를 충분히 다듬은 뒤** 생성을 누르세요. 아무거나 막 시도하면 한도가 5분 만에 끝납니다.
</Callout>

## 저작권 주의사항

이미지 도구마다 약관이 다릅니다. 강의용 데모는 무료로 자유롭게 써도 되지만, **상업 사용은 약관을 한 번 확인**하세요.

| 도구 | 상업 사용 |
| --- | --- |
| Nano Banana (Gemini) | 일반적으로 가능, 단 Google 약관 확인 |
| Bing Image Creator | **개인 사용만 가능**, 상업 사용 제한 |
| Ideogram | 유료 플랜은 상업 사용 가능, 무료 플랜은 제한적 |

<Callout type="warning">
실제로 앱을 운영해서 돈을 받는 단계가 되면, 로고는 **유료 플랜에서 다시 생성**하거나 디자이너에게 의뢰하는 게 안전합니다.
</Callout>

## 자주 묻는 질문

### Q. "이미지 크기는 어떻게 정해요?"

대부분 도구가 정사각형(1:1)이 기본입니다. 로고는 정사각형이 좋고, 배너는 16:9 비율로 만들어 달라고 프롬프트에 명시하면 됩니다.

### Q. "한글 프롬프트 / 영어 프롬프트 중 뭐가 나아요?"

영어 프롬프트가 보통 결과가 더 좋습니다. 한국어로 적은 뒤 ChatGPT에 "이걸 이미지 생성용 영어 프롬프트로 바꿔 줘"라고 부탁하는 게 가장 효율적입니다.

### Q. "결과 이미지를 어떻게 저장해요?"

대부분 마우스 우클릭 → "이미지 저장"으로 됩니다. 도구마다 다운로드 버튼이 따로 있는 경우도 있습니다.
