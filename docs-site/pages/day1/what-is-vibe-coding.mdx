# 바이브 코딩이란?

## 한 줄 정의

> **바이브 코딩** 은 코드를 직접 한 줄 한 줄 타이핑하는 대신, AI에게 자연어로 지시해서 앱을 만드는 방식입니다.

"바이브(vibe)"는 우리말로 옮기면 "느낌·분위기" 정도가 됩니다. 직역하면 "느낌으로 코딩하기"입니다. 마치 디자이너가 "이런 느낌의 포스터 만들어 주세요"라고 하듯, "이런 느낌의 앱 만들어 줘"라고 시키는 거라고 생각하면 쉽습니다.

## 옛날 코딩 vs 바이브 코딩

| 옛날 코딩 (전통적) | 바이브 코딩 (AI 시대) |
| --- | --- |
| 프로그래밍 언어를 배워야 함 | 한국어로 시키면 됨 |
| 문법 외워야 함 | 무엇을 만들지만 설명하면 됨 |
| 한 줄씩 직접 타이핑 | AI가 코드를 써 줌 |
| 에러 나면 직접 디버깅 | 에러를 AI에게 보여 주고 고쳐 달라고 함 |
| 진입 장벽 매우 높음 | 비전공자도 첫 화면을 1시간 안에 띄움 |

<Callout type="info">
**그렇다고 코드를 아예 안 봐도 되는 건 아닙니다.** 깊이 들어가면 결국 코드를 읽거나 수정할 줄 알게 되면 더 강력해집니다. 다만 **시작하는 데 코드 지식이 필요 없을 뿐**입니다.
</Callout>

## 비유로 이해하기

### 비유 1: 음식점 사장님과 주방장

```
직접 코딩 = 사장님이 직접 요리하는 식당
바이브 코딩 = 주방장(AI)에게 "이런 메뉴 내 줘"라고 시키는 사장님
```

사장님이 잘 시키려면 어떤 능력이 필요할까요?

- 손님이 누구인지 안다 (타깃 사용자)
- 어떤 맛을 내고 싶은지 안다 (앱의 컨셉)
- "짠 거 빼고", "양은 적게" 같은 요구를 명확히 한다 (제약)

**바로 이 능력이 바이브 코딩에서 가장 중요한 능력**입니다. 그리고 이건 비전공자도 평소에 잘하는 일입니다.

### 비유 2: 무한히 부려먹을 수 있는 인턴

AI는 24시간 일하고, 잠도 안 자고, 화도 안 내는 인턴이라고 상상해 보세요. 다만 이 인턴은:

- **눈치가 없습니다**. 두루뭉술하게 시키면 엉뚱한 결과를 가져옵니다.
- **기억이 짧습니다**. 매번 처음부터 다시 설명해야 하는 경우가 많습니다.
- **기억이 좋을 때도 있습니다**. 한 대화 안에서는 앞 내용을 기억합니다.

그래서 "**잘 시키는 법**"을 익히는 게 핵심입니다.

## 바이브 코딩으로 진짜 앱을 만들 수 있나요?

네, 만들 수 있습니다. 다만 **"수익이 나는 상용 서비스"** 와 **"내가 만든 데모 앱"** 사이에는 단계가 있습니다.

```
1단계: 화면이 보이는 데모 (Day 2 끝)
   ↓
2단계: 로그인 + 데이터가 저장되는 진짜 앱 (Day 3 끝)
   ↓
3단계: 인터넷에 떠 있고 결제 구조가 붙은 앱 (Day 4 끝)
   ↓
4단계: 사업자 등록 + 마케팅 + 사용자 확보 (강의 이후, 본인 몫)
```

이 강의는 **3단계까지 함께 갑니다**. 4단계는 Day 4 마지막에 가이드만 드립니다.

## 그럼 코딩 공부는 안 해도 되나요?

지금은 안 해도 됩니다. 하지만 만들다 보면 **"이 부분은 직접 손보고 싶다"** 는 욕구가 생길 때가 옵니다. 그때 가서 배워도 늦지 않습니다.

순서는 이렇게 추천합니다:

1. 일단 바이브 코딩으로 **앱 1개를 끝까지 만들어 봅니다**.
2. 그 과정에서 자주 마주친 단어들이 무엇이었는지 적어 둡니다.
3. 그 단어들 위주로 공부합니다. (예: HTML, CSS, JavaScript, 데이터베이스 등)

이 순서가 비전공자에게 가장 효율적입니다. **추상적인 코딩 공부**보다 **구체적인 막힘**이 훨씬 강력한 학습 동기입니다.

## 오늘 첫 체험 — 5분 안에 첫 화면 띄우기

말로 설명만 하면 감이 안 옵니다. 다 같이 Lovable에 한 줄짜리 프롬프트를 던져서, **AI가 진짜로 1~2분 안에 화면을 만들어내는 모습**을 본인 노트북에서 직접 보겠습니다.

자세한 진행은 마지막 챕터 [노코드 앱 빌더 미리보기](/day1/no-code-builder)에서 다 같이 따라 합니다. 지금은 **"내가 직접 만든 화면이 곧 나오겠구나"** 정도만 기대하시면 됩니다.
