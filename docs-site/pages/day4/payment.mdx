# 결제 도구 체험

## 이 챕터의 목표

본인 앱에 **결제 버튼 1개**를 붙여 봅니다. 진짜 돈은 받지 않고, **구조를 손으로 만져 보는** 게 목표입니다.

<Callout type="warning">
**실제 카드 입력 절대 금지.** 어떤 도구에서도 진짜 카드 정보를 넣지 마세요. 토스페이먼츠는 반드시 **테스트 모드 키**만 사용합니다.
</Callout>

## 결제 도구 4가지 비교

| 도구 | 진입 장벽 | 사업자 필요 | 한국 사용자 친화 | 추천 시나리오 |
| --- | --- | --- | --- | --- |
| **Gumroad** | 매우 낮음 | **불필요** | 중간 (해외 카드 위주) | 디지털 상품 빠른 판매 |
| **Stripe Payment Links** | 낮음 | 한국 사업자등록 필요 | 중간 | 글로벌 + 국내 |
| **Lemon Squeezy** | 낮음 | 불필요 (대신 처리) | 중간 | 글로벌 SaaS 시작 |
| **토스페이먼츠** | 중간 | 사업자등록·통신판매업 필요 | **매우 높음** | 한국 대상 본격 운영 |

<Callout type="info">
**오늘 강의에서는 Gumroad로 다 같이 따라 합니다.** 사업자 없이도 결제 링크를 만들 수 있어 **가장 빠르게 손으로 만져 볼 수 있는** 도구입니다.
</Callout>

## 실습 1 — Gumroad에 디지털 상품 1개 등록

### 실습 목표

본인 앱과 관련된 가짜 디지털 상품 1개를 Gumroad에 등록하고, 결제 링크를 받아 본인 앱에 버튼으로 붙입니다.

### 준비물

- [Gumroad](https://gumroad.com) 가입 (이메일 또는 Google 가입)
- 본인 앱과 어울리는 상품 컨셉 (예: "FitCheck 프리미엄", "운동 루틴 PDF")

### 단계

1. Gumroad 가입 → 대시보드
2. **"New Product"** 클릭
3. 상품 종류 선택 → **"Digital product"** 가 가장 간단
4. 상품 정보 입력:

| 항목 | 입력값 예시 |
| --- | --- |
| Name | FitCheck 프리미엄 (1년) |
| Price | 9,900원 또는 $9.90 |
| Description | "FitCheck의 모든 기능을 1년 동안 사용할 수 있는 프리미엄 멤버십" |
| Cover image | Day 1에 만든 로고 또는 새 이미지 |
| Content | 임시로 PDF 1장 업로드 (예: 환영 안내) |

5. **Publish** 클릭 → 결제 링크 발급

### 결제 링크 형식

```
https://yourname.gumroad.com/l/fitcheck-premium
```

이 링크를 누르면 결제 화면이 뜹니다. (실제 카드 입력은 하지 마세요)

## 단계 2 — Lovable에 결제 버튼 추가

본인 앱의 메인 화면이나 설정 화면에 결제 버튼을 붙입니다.

### 프롬프트 — 좋은 예

```text filename="결제 버튼 추가 프롬프트"
설정 화면에 "프리미엄 업그레이드" 섹션을 추가해 줘.

- 섹션 제목: "프리미엄으로 업그레이드"
- 가격 표시: 연 9,900원 (큰 글자)
- 혜택 3가지 불릿 (체크 아이콘 + 한 줄씩):
  ✓ 무제한 운동 기록
  ✓ 친구와 공유 기능
  ✓ 통계 리포트 다운로드
- "지금 결제하기" 버튼 (큰 강조 버튼, 짙은 주황 배경)
  - 클릭 시 새 탭에서 다음 URL 열기:
    https://yourname.gumroad.com/l/fitcheck-premium

스타일:
- 카드 형태, 둥근 모서리, 그림자 살짝
- 다른 설정 항목과 일관된 디자인
```

이 한 방으로 결제 버튼이 들어갑니다. 클릭하면 새 탭에서 Gumroad 결제 페이지가 열립니다.

### 결제 흐름 확인 — 단계별

```
① 본인 앱(배포 URL) 접속 → 로그인
② 설정 화면 → "프리미엄 업그레이드" 섹션 보이는지 확인
③ "지금 결제하기" 클릭 → Gumroad 결제 화면이 새 탭에 뜨는지 확인
④ 여기까지만 — 실제 카드 입력 X
```

## 실습 2 — 토스페이먼츠 테스트 모드 (선택)

토스페이먼츠는 한국 사용자에게 가장 친화적이지만 **사업자 등록이 필요**합니다. 강의에서는 **테스트 모드**로 결제창만 띄워 봅니다. 테스트 모드 키는 사업자 없이도 발급받을 수 있습니다.

### 단계별 따라 하기

```
① https://developers.tosspayments.com 접속 → 가입
② 테스트 모드 활성화 → 테스트용 클라이언트 키 발급
③ 키를 메모장에 복사
④ Lovable에 아래 프롬프트 붙여넣기 → Enter
⑤ "1,000원 결제하기" 버튼 클릭 → 토스 결제창 떠야 함 (카드 입력 X)
```

```text filename="토스페이먼츠 테스트 결제창 프롬프트"
"한국 결제 테스트" 화면을 추가해 줘.
- 토스페이먼츠 SDK를 사용
- 테스트 모드의 클라이언트 키를 환경 변수로 관리
- "1,000원 결제하기" 버튼 클릭 시 토스 결제창 띄움
- 카드/계좌이체 선택 가능
- 성공 시 "결제 성공 (테스트)" 메시지

테스트 모드이므로 실제 청구는 발생하지 않음.
```

<Callout type="error">
**라이브 모드는 절대 활성화하지 마세요.** 라이브 모드는 진짜 결제가 일어납니다. **테스트 모드 키만** 사용합니다. 클라이언트 키 이름이 `test_ck_` 로 시작해야 안전합니다.
</Callout>

## 실습 3 — Stripe Payment Links (글로벌, 가입까지만)

글로벌 사용자를 받을 계획이 있다면 [Stripe](https://stripe.com)가 가장 강력합니다. **Stripe는 2020년부터 한국 정식 진출**하여, 한국 개인사업자/법인으로도 가입할 수 있습니다.

### 특징

- 한국 사업자등록증·신분증·정산 계좌가 있으면 가입 가능합니다.
- 한국 발행 카드도 결제할 수 있고, 원화 결제도 지원합니다.
- 글로벌 사용자(170개국 이상) 대상 운영에 강합니다.

### 가입까지만 시도

```
① https://stripe.com 접속, 가입
② Dashboard 접속
③ Payment Links 메뉴 → 새 링크 생성
④ 상품 정보 + 가격 입력 → 링크 발급까지만
```

실제로 결제를 받으려면 사업자 정보 인증 단계까지 통과해야 합니다.

## 결제 도구 선택 가이드

본인 앱의 상황에 따라 다릅니다.

```
한국 사용자 + 사업자 있음 → 토스페이먼츠
한국 사용자 + 사업자 없음 → Gumroad (디지털 상품)
글로벌 사용자 + 사업자 있음 → Stripe
글로벌 SaaS 시작 (사업자 없음) → Lemon Squeezy
```

## 자주 묻는 질문

### Q. "Gumroad는 한국 사용자가 살 수 있어요?"

네, **카드 결제**는 한국 사용자도 가능합니다. PayPal도 됩니다. 다만 결제 화면이 영어이고, 환율 처리가 약간 어색할 수 있습니다.

### Q. "환불은 어떻게 처리해요?"

- **Gumroad**: 대시보드에서 직접 환불 클릭. 자동 처리.
- **토스페이먼츠**: PG사 관리자 페이지에서 처리. 수수료 일부 환불 안 됨.
- **Stripe**: 대시보드에서 환불 처리. 자동.

환불 정책은 **약관에 미리 명시**해야 합니다.

### Q. "정산은 언제 들어와요?"

- **Gumroad**: 매주 금요일 정산 (Stripe Connect 또는 PayPal). 한국 송금은 PayPal 수령 제한이 있을 수 있어 사전 확인 필요.
- **토스페이먼츠**: 매일/매주 정산 (선택). 보통 거래 후 1~3 영업일.
- **Stripe**: 매일/매주 (계정 설정에 따라).

### Q. "수수료는 얼마예요?"

- **Gumroad**: 거래당 **10% (flat)**. 별도 고정 수수료 없음.
- **토스페이먼츠**: 카드 2.5% / 계좌이체 1.5% 정도
- **Stripe (한국)**: 국내 카드 약 **3.4% + ₩300**, 해외 카드는 별도 (약 +1.5%p)

수수료 차이가 크지만, **Gumroad는 사업자 등록 없이 시작 가능**한 게 가장 큰 장점입니다.
