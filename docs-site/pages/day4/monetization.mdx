# 수익화 구조 이해

## 이 챕터의 목표

본인 앱이 **어떤 방식으로 수익을 낼 수 있는지** 4가지 모델로 분류해서 이해합니다. 그리고 본인 앱이 어디에 가까운지 결정합니다.

## 수익화 모델 4가지

### 1. 일회성 결제 (One-time Purchase)

한 번 결제하면 평생 또는 일정 기간 사용.

| 항목 | 설명 |
| --- | --- |
| 비유 | 책 한 권 사기 |
| 예시 상품 | 디지털 템플릿, 강의 영상, 전자책, 일회성 도구 |
| 진입 장벽 | **낮음** (가입 부담이 적음) |
| 안정성 | 낮음 (한 번 받고 끝) |
| 적합한 앱 | 한 번 쓰고 결과물만 받으면 되는 도구 |

### 2. 구독 (SaaS - Subscription)

매달/매년 정기 결제.

| 항목 | 설명 |
| --- | --- |
| 비유 | 넷플릭스, 헬스장 멤버십 |
| 예시 상품 | 가계부 앱, 일정 관리 도구, 노션, AI 서비스 |
| 진입 장벽 | **높음** (가입에 부담) |
| 안정성 | **매우 높음** (매달 자동 결제) |
| 적합한 앱 | 사용자가 매일/매주 쓰는 도구 |

<Callout type="info">
**개인 메이커가 가장 도전하기 좋은 모델은 구독입니다.** 100명만 월 5,000원씩 내면 월 50만 원입니다. 단, 첫 100명 모으는 게 어렵습니다.
</Callout>

### 3. 광고 (Ads)

콘텐츠는 무료, 광고로 수익.

| 항목 | 설명 |
| --- | --- |
| 비유 | 네이버 블로그, 유튜브 |
| 예시 상품 | 콘텐츠 사이트, 정보 모음 앱, 커뮤니티 |
| 진입 장벽 | **매우 낮음** (사용자 무료) |
| 안정성 | 낮음 (트래픽 의존) |
| 적합한 앱 | **월 방문자 1만 이상**일 때만 의미 있음 |

<Callout type="warning">
**초기엔 비추천**입니다. 월 방문자 1만 명 이상일 때 의미 있는 수익이 시작됩니다. 그 전엔 결제 모델이 빠른 검증이 됩니다.
</Callout>

### 4. 수수료 / 중개 (Commission / Marketplace)

두 사용자 사이의 거래에서 수수료.

| 항목 | 설명 |
| --- | --- |
| 비유 | 우버, 에어비앤비, 당근마켓 |
| 예시 상품 | 거래 플랫폼, 매칭 서비스 |
| 진입 장벽 | 매우 높음 (양면 시장 구축) |
| 안정성 | 매우 높음 (한번 자리 잡으면 강력) |
| 적합한 앱 | **이미 사용자 1,000명 이상**인 플랫폼 |

<Callout type="warning">
**처음 만드는 앱에는 부적합**합니다. 양면 시장(공급자 + 수요자)을 동시에 모아야 해서 매우 어렵습니다. 강의 결과물로는 권장하지 않습니다.
</Callout>

## 본인 앱은 어디에 가까운가?

짝꿍과 1분씩 토론해 보세요.

- 사용자가 **한 번 쓰고 끝**인가? → 일회성
- 사용자가 **매주 / 매일** 쓰는가? → 구독
- 사용자가 **공짜로** 와야 의미 있는가? → 광고 (단, 트래픽 확보 후)
- **두 종류의 사용자**가 만나야 하는가? → 수수료/중개 (어려움)

### 본인 앱별 추천 모델 예시

| 앱 | 추천 모델 | 이유 |
| --- | --- | --- |
| FitCheck (운동 기록) | 구독 (개인) 또는 일회성 (헬스장) | 매일 쓰는 도구 |
| 동네 식당 모음 | 광고 (식당 사장) | 사용자는 무료여야 의미 있음 |
| 가계부 | 구독 또는 일회성 | 매일 쓰는 도구 |
| 독서 메모 | 일회성 | 가벼운 사용 |
| 공구 모임 | 수수료 (어려움) 또는 일회성 (운영자용) | 양면 시장은 어려움 |

## 가격은 얼마가 적절한가?

처음에 자주 하는 실수: **가격을 너무 낮게 책정**.

### 가격 결정의 기본 공식

```
사용자가 받는 가치 ≫ 가격
```

비싸 보이는 가격이 사용자 입장에서는 **시간을 절약하는 비용**이라면 기꺼이 냅니다.

### 가격 예시 (한국 인디 메이커 기준)

| 앱 종류 | 일반적인 가격대 |
| --- | --- |
| 단순한 도구 | 일회성 9,900원 ~ 19,900원 |
| 가벼운 SaaS | 월 3,000원 ~ 9,900원 |
| 본격 SaaS | 월 9,900원 ~ 29,000원 |
| 전문가용 도구 | 월 30,000원 ~ 100,000원 |
| 강의·전자책 | 일회성 29,000원 ~ 99,000원 |

<Callout type="info">
**처음엔 비싼 게 낫습니다.** 너무 싸면 가치가 없는 것처럼 보입니다. 무료 + 유료(2-tier) 또는 1만 원짜리 상품이 가장 자주 팔립니다.
</Callout>

## "진짜 돈을 받으려면" — 한국 기준 행정 흐름

이 부분은 가이드만 드립니다. 강의 시간 안에는 끝나지 않습니다.

### 1단계 — 사업자 등록

- **홈택스 온라인** 5분 (https://hometax.go.kr)
- 일반 과세자 또는 **간이 과세자** 선택 (연 매출 8천만 원 미만이면 간이 가능)
- **즉시 발급**

### 2단계 — 통신판매업 신고

- **정부24** 온라인 (https://gov.kr)
- 무료~소액 수수료
- 약 1~2주 처리

### 3단계 — 결제 PG 가입

- **토스페이먼츠** 또는 **KG이니시스**가 한국 사용자에게 친화적
- 사업자등록증 + 통신판매업 신고증 필요
- 정산 통장 연결

### 4단계 — 약관·환불정책·개인정보처리방침

- 전자상거래법상 **반드시 명시**해야 함
- 무료 템플릿 활용 가능 (예: 위시켓·로앤컴퍼니 템플릿)
- 변호사 검토 권장 (월 단위 구독은 특히 중요)

### 5단계 — 정산

- 매주~매월 본인 계좌로 입금 (PG 정책에 따라 다름)
- 부가세 신고 (간이 과세자는 연 1회, 일반은 연 2회)
- **세무사 도움 권장** (월 5~10만 원)

<Callout type="warning">
**이 모든 게 강의 범위 밖**입니다. 실제로 시작하시려면 1~2주 정도 행정 시간이 필요합니다.
</Callout>

## 핵심 메시지

> **결제 모듈을 붙이는 건 가장 쉬운 단계입니다.**
> **사람들이 이 앱을 발견하고, 가입하고, 돈 낼 가치를 느끼게 만드는 게 진짜 어려운 단계입니다.**

그래서 다음 챕터에서는 결제뿐 아니라 **랜딩페이지·SEO·도메인**까지 함께 봅니다. 결제 모듈만큼 중요합니다.

## 자주 묻는 질문

### Q. "사업자 등록 안 하고 돈 받을 수 있어요?"

일정 금액 이하의 비정기 수익은 가능하지만, **권장하지 않습니다.** 정식으로 운영하려면 등록 필수입니다. 미등록 상태로 매출이 일정 수준 넘으면 세무 문제가 생길 수 있습니다.

### Q. "부가세는요?"

연 매출 8천만 원 미만이면 **간이 과세자**가 가능합니다. 부가세 부담이 크지 않고 절차도 간단합니다. 자세한 건 세무사 상담을 권장합니다.

### Q. "앱이 잘 안 되면 환불해 줘야 하나요?"

전자상거래법상 디지털 상품도 환불 정책 명시 의무가 있습니다. 약관에 **"디지털 상품의 환불 가능 조건"** 을 명확히 적어 두세요. (예: "사용 시작 전까지 100% 환불 가능, 사용 시작 후엔 환불 불가" 등)

### Q. "광고로 돈 벌고 싶어요"

월 방문자 1만 명 이상은 되어야 의미 있는 수익이 시작됩니다. 그 전엔 **결제 모델이 더 빠른 검증**입니다. 광고는 트래픽이 쌓인 다음 추가하세요.
