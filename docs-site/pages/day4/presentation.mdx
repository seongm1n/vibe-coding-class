# 발표하기 (1인 1~2분)

## 이 챕터의 목표

본인이 4일간 한 일을 **1~2분 안에** 정리해서 다른 수강생들과 공유합니다.

## "완성도"가 아니라 "시도"를 공유합니다

발표는 평가가 아닙니다. 잘 만들었나/못 만들었나가 아니라, **무엇을 시도했고 무엇을 배웠는가**를 나누는 시간입니다.

- 끝까지 못 만든 사람도 발표합니다.
- "여기까진 됐는데 이 부분에서 막혔습니다" 도 좋은 발표입니다.
- 오히려 **막힌 곳 공유**가 다른 수강생에게 가장 도움이 됩니다.

## 발표 형식 — 3가지만 말해 주세요

```
1) 누구를 위한 앱인지 (한 줄)
2) 4일간 무엇을 시도했는지 (어디까지 됐고 어디서 막혔는지)
3) 다음에 뭘 더 해 보고 싶은지 (한 줄)
```

이 3가지면 1~2분이 자연스럽게 채워집니다.

## 발표 스크립트 템플릿

다음 스크립트에 본인 내용을 채워 넣으면 바로 발표 준비 끝입니다.

```text filename="발표 스크립트 템플릿"
안녕하세요, [본인 이름]입니다.

[1. 누구를 위한 앱인지]
저는 [타깃 사용자, 한 줄]를 위한 [앱 이름]을 만들었습니다.
이 사람들이 [어떤 문제]를 겪고 있어서, [앱이 해결하는 방식]으로 도와줍니다.

[2. 4일간 무엇을 시도했는지]
4일 동안 다음을 시도했습니다.
- Day 1에는 [이미지 / 음성 / 영상 등]을 만들어 봤고
- Day 2에는 PRD를 작성하고 첫 화면을 띄웠습니다.
- Day 3에는 회원가입과 데이터 저장까지 됐고 / 또는 어디서 막혔습니다.
- Day 4에는 결제 버튼과 랜딩페이지까지 추가했습니다.

[현재 상태]
지금은 [Vercel URL]에서 접속 가능합니다.
(또는: 배포는 못 했지만 Lovable에서는 동작합니다.)

[3. 다음에 뭘 해 보고 싶은지]
다음에는 [구체적인 한 가지]를 더 해 보고 싶습니다.

감사합니다.
```

## 발표 시 보여 줄 것

발표 중에 **본인 화면을 띄울 시간**이 있을 수 있습니다. 다음을 미리 준비하세요.

1. **본인 앱 URL** — 모니터에 띄울 준비
2. **본인 랜딩페이지 URL** (있으면) — 카톡 미리보기 캡처도 좋음
3. **(선택) 핵심 화면 1~2장 캡처** — 만일을 대비

## 발표 예시

### 예시 1 — 끝까지 잘 된 사람

```
안녕하세요, 김OO입니다.

저는 헬스장 자주 다니는 30대 직장인을 위한 운동 기록 앱
"FitCheck"을 만들었습니다.

운동 기록을 매번 까먹어서 30초 안에 한 줄로 기록할 수 있게 만들었어요.

4일간:
- Day 1엔 로고를 Ideogram으로 만들었고
- Day 2엔 PRD를 ChatGPT랑 같이 정리하고 Lovable로 첫 화면 띄웠고
- Day 3엔 회원가입이랑 데이터 저장까지 진짜로 됐고
- Day 4엔 Gumroad로 결제 버튼 붙이고 랜딩페이지까지 만들었습니다.

지금 fitcheck-app.vercel.app에서 접속 가능합니다. 혹시 가입해 주시면
오늘 안에 친구 5명 만드는 게 목표예요.

다음엔 친구랑 운동 일정 공유하는 기능을 더 해 보고 싶어요.

감사합니다.
```

### 예시 2 — 중간에 막힌 사람도 OK

```
안녕하세요, 박OO입니다.

저는 동네 식당 영업시간 모음 앱을 만들고 싶었어요.
점심 시간에 매번 어디 갈지 고민하는 직장인을 위해서요.

4일간:
- Day 1엔 LLM 비교가 가장 인상 깊었고
- Day 2엔 PRD를 만들고 Lovable로 메인 화면 띄웠는데
- Day 3에 Supabase 연결하다가 환경 변수 부분에서 시간이 다 갔어요.
- 결국 회원가입은 됐는데 데이터 저장은 못 끝냈습니다.

근데 이 강의에서 가장 큰 수확은 "이걸 비전공자가 할 수 있다"는 감각인 것 같아요.

다음엔 막힌 데이터 저장 부분 천천히 다시 시도해 보겠습니다.

감사합니다.
```

이런 발표가 오히려 **다른 사람에게 용기를 줍니다**. 솔직한 게 더 가치 있습니다.

## 시간 관리

- 1인 1~2분 엄수
- 강사가 타이머를 들고 있을 수 있습니다
- **2분 넘기면 짧게 끊습니다** (다른 사람 발표 시간 확보)

## 발표 후 — URL 일괄 공유

발표가 끝나면 다음을 한 곳에 모읍니다.

- 본인 앱 URL
- 본인 랜딩페이지 URL (있으면)
- 본인 GitHub 레포 URL (공개로 했다면)

공유 방법은 다음 중 하나입니다.

- 강의 카톡방
- [Padlet](https://padlet.com) 같은 협업 메모 보드
- 강사가 안내하는 별도 폼

이렇게 모으면 강의 후에도 서로의 진행 상황을 볼 수 있습니다.

## 박수와 사진

- 모든 발표 후엔 **박수**로 마무리합니다.
- **단체 사진** 한 장 찍어 두면 강의 후에도 따뜻한 기억으로 남습니다.

## 자주 묻는 질문

### Q. "발표가 부담스러워요"

안 해도 됩니다. **본인 의사 100% 존중**합니다. 다만 한 줄이라도 말씀하시면 그게 다음 발표자에게 용기가 됩니다.

### Q. "내 앱이 부끄러운데요?"

전부 그렇게 느낍니다. 4일 만에 만든 앱이 멋진 게 더 이상합니다. **시도한 것만으로 이미 충분**합니다.

### Q. "발표 후 피드백 받아도 되나요?"

원하시면 발표 끝에 "혹시 피드백 있으면 환영합니다"라고 한 줄 덧붙이세요. 강의 후 카톡방에서 피드백이 자연스럽게 옵니다.
