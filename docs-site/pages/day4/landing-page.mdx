# 랜딩페이지 + SEO + 도메인

## 이 챕터의 목표

본인 앱을 **소개하는 1페이지**(랜딩페이지)를 만들고, 검색에 잘 걸리도록 SEO 기초를 적용합니다. 시간이 남으면 본인 도메인(.com)을 연결합니다.

## "랜딩페이지"가 뭔가요?

**랜딩페이지(Landing Page)** 는 사람이 처음 도착하는(land) 한 페이지짜리 소개 사이트입니다. 본 앱과는 별개입니다.

```
랜딩페이지 (1장)
   ↓ "지금 시작하기" 버튼
실제 앱 (회원가입 → 메인 화면)
```

비유하자면, **광고 전단지**입니다. 식당 앱이라면 "이 앱은 무엇을 하고, 누구한테 좋고, 어떻게 시작할 수 있는지"를 한 장에 담습니다.

## 랜딩페이지의 5가지 구성 요소

좋은 랜딩페이지는 거의 다음 5가지 순서로 구성됩니다.

```
1. 헤드라인 (한 줄로 무엇인지)
   ↓
2. 문제 제시 (지금 사용자가 겪는 불편)
   ↓
3. 해결 방법 (이 앱이 어떻게 해결하는지)
   ↓
4. 스크린샷 (실제 화면 모습)
   ↓
5. CTA (Call To Action — 가입/결제 버튼)
```

## 실습 — Lovable에 랜딩페이지 추가

기존 Lovable 프로젝트에 새 페이지로 랜딩을 추가합니다. 또는 별도 프로젝트로 만들어도 됩니다.

### 프롬프트 — 좋은 예

```text filename="✅ 랜딩페이지 생성 프롬프트"
이 앱을 소개하는 1페이지 랜딩페이지를 별도 경로(/landing 또는 /)로 만들어 줘.

구성 (위에서 아래로):

[1. Hero 섹션]
- 큰 헤드라인: "30초면 끝나는 운동 기록"
- 부제: "복잡한 헬스장 앱에 지친 분들을 위한 가장 심플한 운동 일기"
- 우측 또는 아래에 앱 스크린샷 1장 (실제 메인 화면 캡처처럼)
- 큰 CTA 버튼: "무료로 시작하기" → 회원가입 화면(/signup)으로 연결

[2. 문제 섹션]
- 섹션 제목: "이런 적 있으세요?"
- 3개 카드, 각 카드에 짧은 멘트:
  · "노트에 적었는데 어디 갔는지 모르겠다"
  · "헬스장 앱 너무 복잡해서 안 쓰게 된다"
  · "운동 기록을 친구랑 공유하고 싶다"

[3. 해결 섹션]
- 섹션 제목: "FitCheck로 해결합니다"
- 3개 카드, 각 카드에 (아이콘 + 제목 + 한 줄 설명):
  · ⚡ 30초 입력: 종목·무게·횟수 3개만 입력하면 끝
  · 📊 한눈에 보기: 이번 주 운동을 요일별 카드로
  · 🤝 친구와 공유: 카톡으로 이번 주 요약 공유

[4. 스크린샷 섹션]
- 섹션 제목: "이렇게 작동합니다"
- 가로 3컷 (모바일 화면 mockup): 메인 → 입력 → 기록 보기

[5. 가격 섹션]
- 섹션 제목: "가격"
- 무료 / 프리미엄 두 카드 비교
  - 무료: 운동 기록 무제한, 본인만 보기
  - 프리미엄 9,900원/년: + 친구 공유 + 통계 리포트
- 프리미엄 카드의 큰 버튼: "프리미엄 시작" → Gumroad 링크

[6. 마지막 CTA]
- 큰 헤드라인: "오늘부터 시작하세요"
- 부제: "30초면 첫 기록을 남길 수 있습니다"
- 큰 버튼: "무료로 시작하기"

[Footer]
- 작은 글씨로: "© 2026 [본인 앱 이름] · 문의: [본인 이메일]"

스타일:
- 모던, 깔끔, 둥근 모서리
- 색: 흰 배경 + 짙은 주황 강조
- 모바일 우선
- 섹션 사이 여백 충분히

언어: 한국어
```

이 프롬프트 한 방으로 랜딩페이지가 완성됩니다.

## SEO 기초 — 검색에 걸리게 하기

### "SEO"가 뭔가요?

**SEO (Search Engine Optimization)** 는 우리말로 옮기면 "검색 엔진 최적화"입니다. 구글·네이버에서 검색했을 때 본인 사이트가 위쪽에 나오게 하는 작업입니다.

### 가장 중요한 3가지

#### 1. 페이지 제목 (title)

브라우저 탭에 보이는 제목입니다. 검색 결과에도 큰 글자로 나옵니다.

```text filename="좋은 title 예시"
30초면 끝나는 운동 기록 — FitCheck
```

```text filename="나쁜 title 예시"
홈
```

**60자 이내**, **앱 이름 + 한 줄 설명** 형식이 가장 좋습니다.

#### 2. 메타 설명 (meta description)

검색 결과에서 제목 아래 작게 나오는 설명입니다.

```text filename="좋은 description 예시"
운동 기록을 30초 안에 남기고 친구와 공유하는 가장 단순한 헬스 일기 앱. 무료로 시작하세요.
```

**150자 이내**, **사용자가 클릭할 만한** 짧은 소개를 적습니다.

#### 3. 헤드라인에 검색어 포함

타깃 사용자가 검색할 만한 단어를 헤드라인에 자연스럽게 넣습니다.

```
검색어: "운동 기록 앱"
헤드라인: "30초면 끝나는 운동 기록 앱"  ← 자연스럽게 포함
```

### Lovable에 SEO 적용

```text filename="SEO 적용 프롬프트"
랜딩페이지에 다음 SEO 메타 정보를 추가해 줘:

<head> 안에:
- <title>30초면 끝나는 운동 기록 — FitCheck</title>
- <meta name="description" content="운동 기록을 30초 안에 남기고 친구와 공유하는 가장 단순한 헬스 일기 앱. 무료로 시작하세요." />
- <meta property="og:title" content="..." />
- <meta property="og:description" content="..." />
- <meta property="og:image" content="(앱 로고 또는 스크린샷 URL)" />
- <meta name="twitter:card" content="summary_large_image" />

og:image는 카톡·페이스북·트위터에서 링크 미리보기로 보입니다.
```

<Callout type="info">
**og: 태그**는 카톡으로 링크를 보냈을 때 미리보기 이미지·제목이 어떻게 보일지 결정합니다. 안 넣으면 카톡 미리보기가 비어 있어 보여 클릭율이 떨어집니다.
</Callout>

## 도메인 연결 (선택)

기본 Vercel 도메인(`xxx.vercel.app`)도 충분하지만, 본인 도메인(`fitcheck.com` 같은)이 있으면 훨씬 전문적으로 보입니다.

### 무료 옵션 — Vercel 기본 도메인 그대로

```
https://fitcheck-app.vercel.app
```

**아무 비용 없습니다.** 강의 데모 단계에서는 이걸로 충분합니다.

### 유료 옵션 — 본인 .com 도메인

#### 1단계: 도메인 구매

| 도메인 등록업체 | 가격 (.com 기준 연) | 특징 |
| --- | --- | --- |
| [가비아](https://www.gabia.com) | 약 16,500원 | 한국어, 한국 카드 결제 |
| [후이즈](https://www.whois.co.kr) | 약 17,000원 | 한국어 |
| [Cloudflare](https://www.cloudflare.com) | $9~10 (약 13,000원) | 영문, 가장 저렴 |
| [Namecheap](https://www.namecheap.com) | $10~15 | 영문, 글로벌 |

#### 2단계: Vercel에 도메인 연결

1. Vercel 프로젝트 → **Settings** → **Domains**
2. **"Add Domain"** → 본인 도메인 입력 (예: `fitcheck.com`)
3. Vercel이 안내하는 **DNS 레코드** 정보를 도메인 등록업체 관리자 페이지에 입력
   - A 레코드: `76.76.21.21` (Vercel 안내값)
   - 또는 CNAME: `cname.vercel-dns.com`
4. 5분~24시간 대기 → 자동 연결 + HTTPS 인증서 자동 발급

<Callout type="info">
**시간이 부족하면 도메인은 데모만** 보여 줍니다. 강의 후 천천히 시도해도 됩니다.
</Callout>

### 도메인 작명 팁

- **짧게** (10글자 내외)
- **외우기 쉽게** (영어 단어 또는 발음 쉬운 한국어 영문 표기)
- **앱 이름 그대로** 또는 **앱 이름 + 짧은 단어** (예: `fitcheck.app`)

`.com`이 가장 일반적이지만, `.app`, `.io`, `.kr`도 좋습니다.

## 미리보기로 카톡 공유 테스트

랜딩페이지 URL을 본인 카톡(나에게 보내기)에 한 번 보내 보세요.

```
[ 잘 된 예 ]
링크 미리보기에 로고/스크린샷 + 제목 + 한 줄 설명이 보임

[ 잘못된 예 ]
링크 미리보기가 텅 비어 있거나 일부 정보만 보임
→ og: 태그를 다시 점검해야 함
```

## 자주 묻는 질문

### Q. "랜딩페이지를 본 앱과 같은 도메인에 둬요, 따로 둬요?"

처음엔 **같은 도메인의 다른 경로**(예: `fitcheck.com` 첫 페이지가 랜딩, `fitcheck.com/app`이 실제 앱)가 가장 단순합니다.

### Q. "한국어 도메인은 어때요?"

`헬스기록.kr` 같은 한국어 도메인도 가능하지만, **공유할 때 글자 깨짐**이 자주 발생해 비추천입니다. 영문 도메인을 권장합니다.

### Q. "SEO가 효과 있을 때까지 얼마나 걸려요?"

새 도메인은 **2~6개월** 정도 걸립니다. SEO는 장기전입니다. 단기 사용자 확보는 SNS·커뮤니티 공유가 더 빠릅니다.

### Q. "구글에 빠르게 등록하고 싶어요"

[Google Search Console](https://search.google.com/search-console)에 본인 도메인을 등록하면 **빨리 색인**됩니다. Vercel 무료 도메인도 등록 가능합니다.
