# 화면 다듬기

## 이 챕터의 목표

만들어진 화면에 본인 색깔을 입히고, 모바일에서 잘 보이는지 점검합니다. 디자인을 완벽하게 만드는 것보다 **일관성**을 잡는 게 목표입니다.

## 다듬어야 할 4가지

| 항목 | 무엇을 점검하는가 |
| --- | --- |
| 1. 로고·이미지 | Day 1에 만든 로고가 적용됐는가 |
| 2. 색상 통일 | 강조색 1개로 일관되는가 |
| 3. 폰트 일관성 | 글자 크기 단계가 일정한가 |
| 4. 모바일 미리보기 | 모바일에서 화면이 안 깨지는가 |

## 1. 로고 / 이미지 적용

Day 1에 만들었던 로고 PNG가 있으면 Lovable에 적용합니다.

### 단계

1. 채팅창에서 **이미지 업로드** 버튼 (또는 드래그&드롭) → 로고 파일 업로드
2. 다음과 같이 시킵니다.

```text filename="로고 적용 프롬프트"
방금 업로드한 이미지를 앱 상단 로고로 사용해 줘.
- 헤더 왼쪽에 로고 + 앱 이름 텍스트 배치
- 로고 크기: 32px
- 모바일에서도 잘리지 않게
- 다크모드일 땐 자동으로 색이 반전되거나 배경이 들어가게
```

샘플 데이터 안에 들어가는 이미지(예: 식당 사진, 책 표지, 운동 동작 이미지)도 다음과 같이 시킬 수 있습니다.

```text filename="샘플 이미지 프롬프트"
샘플 식당 카드의 이미지를 Unsplash의 음식 사진으로 자동 채워 줘.
다양한 메뉴(한식/일식/양식)가 골고루 섞이게.
```

<Callout type="info">
**Unsplash·Pexels 같은 무료 이미지 사이트**에서 자동으로 이미지를 끌어오는 게 Lovable의 기본 동작입니다. 별도로 사진을 준비하지 않아도 화면이 그럴듯해집니다.
</Callout>

## 2. 색상 통일

PRD에 정한 강조색 1개로 일관되게 만듭니다. **색이 5개 이상 섞여 있으면** 화면이 어수선해 보입니다.

### 단계

```text filename="색상 통일 프롬프트"
앱 전체 색상을 다음 팔레트로 통일해 줘:
- 배경: 흰색 (#ffffff), 다크모드는 #0f0f0f
- 기본 텍스트: 진한 회색 (#222222), 다크모드는 #eaeaea
- 보조 텍스트: 회색 (#666666)
- 강조색 (버튼·링크·강조 표시): #c8401a
- 경고/위험: 빨강 (#d12626)
- 성공: 녹색 (#1f9e4a)

기존 화면에서 위 팔레트에 안 맞는 색은 모두 교체해 줘.
```

<Callout type="info">
**색은 3~5개로 충분합니다.** 배경 1개, 텍스트 2개(기본/보조), 강조색 1개, 그리고 경고/성공 색 1~2개 정도면 됩니다.
</Callout>

## 3. 폰트 일관성

폰트 크기·굵기는 **단계(스케일)** 로 정해 두면 일관됩니다.

### 단계

```text filename="폰트 통일 프롬프트"
글자 크기를 다음 스케일로 정리해 줘:
- 페이지 제목 (h1): 24px, 굵게(bold)
- 섹션 제목 (h2): 20px, 굵게
- 카드 제목 (h3): 16px, 중간 굵기(semibold)
- 본문 (p): 14px, 보통(regular)
- 작은 설명 (small): 12px, 보통, 회색

폰트는 한국어가 잘 보이는 시스템 폰트 또는 Pretendard를 사용해 줘.
줄 간격(line-height)은 본문은 1.6, 제목은 1.3.
```

이렇게 정해 두면 다음에 화면을 추가할 때도 같은 스케일로 만들어 줍니다.

## 4. 모바일 미리보기 점검

Lovable 미리보기 패널 상단에 **모바일/데스크톱 토글**이 있습니다. 둘 다 확인하세요.

### 모바일에서 자주 깨지는 부분

| 문제 | 원인 | 해결 프롬프트 |
| --- | --- | --- |
| 가로 스크롤 발생 | 콘텐츠가 화면보다 넓음 | "모바일에서 가로 스크롤이 안 생기게 모든 콘텐츠가 화면 폭에 맞게" |
| 글자가 잘림 | 글자 크기가 너무 큼 | "모바일에서 제목 24px → 20px로" |
| 버튼이 눌리지 않음 | 버튼이 너무 작음 | "모든 버튼은 최소 높이 44px (손가락 터치 영역 확보)" |
| 입력 폼이 화면 밖 | 키보드가 폼을 가림 | "입력 폼이 화면 중앙보다 약간 위에 오게" |

### 일괄 점검 프롬프트

```text filename="모바일 점검 프롬프트"
모바일 화면(가로 360px)을 가정하고 다음을 모두 점검해 수정해 줘:
- 가로 스크롤이 발생하는 곳은 없는가
- 모든 텍스트가 잘리지 않고 잘 보이는가
- 버튼은 최소 높이 44px, 손가락으로 누르기 편한가
- 화면 양쪽 여백은 16px 이상인가
- 카드 사이 간격은 12~16px인가

문제가 되는 부분을 모두 수정해 줘.
```

## 다크모드도 확인 (선택)

Lovable은 보통 자동으로 다크모드를 만들어 줍니다. 미리보기에서 다크모드 토글을 눌러 확인하세요.

```text filename="다크모드 점검 프롬프트"
다크모드에서 다음을 점검해 줘:
- 흰색 배경이 검은색으로 잘 바뀌었는가
- 텍스트 색이 너무 어둡지 않은가 (배경과 대비 충분한지)
- 강조색은 다크모드에서도 충분히 보이는가
- 그림자나 테두리가 다크모드에 맞게 조정됐는가
```

## 디자인 참고 — 일관성 체크리스트

다음을 하나씩 점검하세요.

- [ ] 같은 종류의 버튼은 같은 색·크기인가
- [ ] 카드 간격이 화면마다 동일한가
- [ ] 둥근 모서리(border-radius)가 동일한가 (예: 12px)
- [ ] 그림자가 모든 카드에 비슷한 강도로 들어가 있는가
- [ ] 한 페이지에 글자 굵기가 너무 많이 섞이지 않았는가 (3~4단계 이내)

## 메시지가 1~2개 남았다면

마지막 메시지는 **"전체 일관성 점검"** 한 방에 쓰는 게 좋습니다.

```text filename="마지막 메시지로 쓸 점검 프롬프트"
지금까지 만든 모든 화면을 다음 기준으로 일괄 점검해 수정해 줘:
1. 색상이 정해진 팔레트(흰 배경/검은 텍스트/짙은 주황 강조)로 통일됐는지
2. 폰트 스케일이 일정한지 (24/20/16/14/12)
3. 둥근 모서리가 12px로 통일됐는지
4. 카드 간격이 16px로 통일됐는지
5. 모바일·데스크톱·다크모드 모두에서 깨진 부분이 없는지

문제 있으면 모두 한 번에 수정해 줘.
```

## 체크포인트

- [ ] 색상이 3~5개로 정리됨
- [ ] 글자 크기 스케일이 일정함
- [ ] 모바일 미리보기에서 가로 스크롤 없음
- [ ] 다크모드도 깨지지 않고 보임
