## 2. "내 앱에 저장될 데이터"가 뭔지 한 줄 정리

내일은 데이터를 진짜로 저장하는 시간입니다. 본인 앱에서 **무엇이 저장되어야 하는지** 한 줄로 적어 오세요.

### 양식

```text filename="저장 데이터 메모 양식"
내 앱: [앱 이름]

저장될 데이터 1종:
- 무엇: [데이터 종류 한 줄]
- 항목: [어떤 항목들이 들어가는가]

예시:
"내 앱: FitCheck
저장될 데이터: 사용자가 입력한 운동 기록
항목: 날짜 / 종목 / 무게 / 횟수 / 메모"
```

### 좋은 예시들

| 앱 | 저장될 데이터 | 항목 |
| --- | --- | --- |
| 운동 기록 일기 | 운동 기록 | 날짜 / 종목 / 무게 / 횟수 / 메모 |
| 동네 식당 모음 | 즐겨찾기 식당 | 식당 ID / 사용자 ID / 별점 / 추가일 |
| 가계부 | 거래 내역 | 날짜 / 금액 / 카테고리 / 메모 / 수입or지출 |
| 독서 메모 | 책 메모 | 제목 / 저자 / 별점 / 한 줄 평 / 인상 깊은 구절 |
| 공구 모임 | 공구 신청 | 공구 ID / 사용자 ID / 수량 / 신청일 |

<Callout type="info">
**한 종류만 정해도 됩니다.** Day 3에서는 "한 종류의 데이터를 저장하고 다시 불러오는" 한 사이클만 끝내는 게 목표입니다.
</Callout>

## 4. (선택) 친구·동료에게 화면 보여 주기

오늘 만든 화면을 누군가에게 보여 주고 다음을 물어 보세요.

```text
"이 화면 보고 어떤 앱인지 짐작 가?"
"여기서 가장 먼저 누를 버튼은 어디야?"
"빠진 게 있다면 뭐야?"
```

## 진도가 빠른 사람용 — 추가 시도

오늘 너무 빠르게 끝났다면 다음을 시도해 보세요.

1. **세 번째 화면 추가** — 예: 설정 페이지, 프로필 페이지
2. **빈 상태(Empty State)** — 데이터가 없을 때 보여 줄 안내 화면
3. **에러 상태(Error State)** — 에러가 났을 때 사용자에게 보여 줄 메시지

```text filename="빈 상태 추가 프롬프트"
"이번 주 기록" 화면에서 운동 기록이 하나도 없을 때
다음과 같은 빈 상태 화면을 만들어 줘:

- 가운데에 친근한 일러스트 1개 (Lovable이 자동 생성)
- "아직 기록이 없어요" 제목
- "오른쪽 위 + 버튼으로 첫 운동을 기록해 보세요" 부제
- 큰 "지금 기록하기" 버튼

빈 상태에서 사용자가 다음에 무엇을 해야 할지 명확하게 보이게.
```
