# 아이디어 점검 — 4일 안에 만들 수 있는 앱인가?

오늘 본격적으로 PRD를 쓰기 전에, 어제 가져온 아이디어가 **4일 안에 비전공자가 만들 수 있는 규모인지** 점검합니다.

<Callout type="info">
**이 챕터가 첫 챕터인 이유**
- 늦게 도착한 분들이 자연스럽게 따라잡을 수 있는 시간
- 어제 아이디어를 안 가져왔거나 다듬고 싶은 분들에게 다시 한 번
- **너무 큰 아이디어는 여기서 잘라내야** PRD/Lovable 단계에서 헛수고를 안 합니다

부담 갖지 마세요. 25분 안에 끝납니다.
</Callout>

## Day 1 핵심 짚고 가기 — 3줄 요약

늦게 오신 분들을 위해. 어제 강의의 핵심만 짧게.

1. **AI 도구는 지금 진짜 잘 됨** — 텍스트 / 이미지 / 음악 / 영상 / 노코드 앱 빌더까지 무료로 가능
2. **"프롬프트 = 지시서"** — AI에게 시킬 때 "누가, 왜, 무엇을, 어떻게"가 잘 정리될수록 결과가 일관됨
3. **오늘부터는 Lovable 본격 사용** — 어제 가입했던 노코드 앱 빌더로 본인 앱 첫 화면 만들기

## 1. 아이디어 없으신 분 — 주변에서 찾는 5가지 질문 (10분)

어제 아이디어를 못 가져왔거나, 가져왔는데 마음에 안 드시면 다음 5가지 질문을 본인에게 던져 보세요. **하나만 걸려도 충분합니다.**

### 질문 1 — 본인 일과를 5분 단위로 적어 보면?

```
오늘 아침:
6:30 일어나서 폰 봄
6:35 출근 준비 시작
7:00 지하철 타고 출발
7:30 카페에서 커피 사면서 어제 운동량 체크 ← 이런 게 자주 반복?
...
```

매일 반복하는 잔일 중 **번거로운 것**이 있다면 그게 곧 앱 아이디어입니다.

### 질문 2 — 가족·동료가 자주 하는 불평이 뭔가요?

```
"또 이거 엑셀로 정리해야 하네..."
"카톡방에서 누가 뭐 시켰는지 또 까먹었어..."
"이번 달 가스비 얼마 나왔지? 아 영수증 어디 갔지..."
```

남의 불평이 본인보다 객관적이라 더 좋은 신호입니다.

### 질문 3 — 본인 폰에 깔린 앱 중 안 쓰는 게 뭔가요?

왜 안 쓸까요? "복잡해서" / "기능이 너무 많아서" / "느려서"가 답이라면, **그 앱의 단순화 버전**이 본인 아이디어가 될 수 있습니다.

### 질문 4 — 본인 직장·가정에서 자주 쓰는 엑셀·노트가 있나요?

엑셀이나 카톡방·메모장으로 매번 정리하는 게 있으면, **그걸 앱으로 만드는 게 가장 쉬운 첫 앱**입니다. 이미 본인이 사용자 + 사용 흐름을 알고 있기 때문.

### 질문 5 — 부모님·할머니·할아버지가 못 쓰시는 앱이 있나요?

기존 앱이 너무 복잡해서 어르신이 못 쓰는 거라면, **단순화 버전**이 좋은 아이디어가 됩니다. (예: "약 먹기 알림 — 큰 글자, 버튼 하나만")

### 5분 안에 아이디어 결정하는 템플릿

위 5가지 중 하나라도 걸렸다면 다음을 한 단락으로 적어 보세요.

```text filename="5분 아이디어 템플릿"
누가 쓸까?       → [본인 / 가족 / 직장 동료 / 지인 1명] 중 한 명
지금 어떻게 풀고 있나? → [메모장 / 엑셀 / 카톡 / 머릿속에만]
앱이 되면 뭐가 좋아질까? → [시간 절약 / 실수 줄이기 / 한눈에 보기]
```

이 3줄이 채워지면 다음 챕터(PRD)로 가도 됩니다.

<Callout type="info">
**그래도 안 떠오르면 백업 PRD 5개 사용** — 강사가 미리 준비한 5가지(동네 식당 모음 / 운동 기록 일기 / 가계부 / 독서 메모 / 공구 모임 정리) 중 하나를 그대로 쓰셔도 OK. **부끄러운 일 아닙니다.** 이 강의는 "독창적인 아이디어 생산"이 아니라 **"워크플로우 손에 익히기"** 가 목표입니다.
</Callout>

## 2. 아이디어 복잡도 가늠 — 4일 안에 가능한가? (10분)

비전공자에게 가장 어려운 게 **"이 아이디어가 4일 안에 가능한지 판단하는 것"** 입니다. 화면을 직접 만들어 본 적이 없으니 감이 없는 게 당연합니다.

다음 신호들로 가늠합니다.

### 🟢 4일 안에 가능 — 청신호

- 화면이 **4~6개 이내**로 다 그려짐
- 사용자가 **혼자 쓰는 앱** (1인 데이터 저장·조회 위주)
- 데이터가 **텍스트·숫자 위주** (사진 1~2장은 OK)
- 외부 서비스 연결이 **1~2개** (Supabase + 결제 링크 1개 정도)
- 흐름이 **단순함**: 홈 → 입력 → 목록 → 상세

위 5가지에 대부분 해당하면 **무난히 가능**합니다.

### 🔴 4일 안에 불가능 — Red Flag 8가지

다음 중 **하나라도 핵심 기능에 들어 있으면** 4일 안에 못 끝냅니다. 다 빼거나, 단순화 버전으로 자르거나, 다른 아이디어로 갈아타세요.

| Red Flag | 예시 | 이유 |
|----------|------|------|
| **실시간 협업 / 동시 편집** | 슬랙·구글닥스 같은 실시간 채팅·편집 | WebSocket 인프라 필요 |
| **복잡한 결제·정산** | 다수 입금자 분할 정산 / 환불·취소 / 정기결제 관리 | 결제 게이트웨이 + 정산 로직이 무거움 |
| **외부 API 5개 이상** | 카카오맵 + 토스 + 카톡 알림 + 네이버 검색 + 인스타 동시 | API 키 관리·에러 핸들링이 압도적 |
| **AI 모델 자체 학습** | 본인 데이터로 GPT 같은 모델 fine-tuning | 시간·비용 모두 강의 범위 밖 (LLM 호출은 OK) |
| **모바일 네이티브 기능** | 블루투스 / 백그라운드 위치 추적 / 카메라 OCR / 푸시 알림 | 웹앱 한계, 네이티브 앱은 별도 과정 |
| **다중 사용자 권한 분리** | 관리자 / 직원 / 고객 3단 권한, 각각 다른 화면 | 인증·권한 흐름이 복잡 |
| **영상·음성 실시간 처리** | 라이브 스트리밍 / 실시간 음성 변환 | 인프라·대역폭 모두 무거움 |
| **ML 기반 추천·검색** | "내 취향에 맞는 식당 자동 추천" | 데이터·모델·평가까지 별도 영역 |

### 🟡 헷갈리는 함정 — 단순한 듯 복잡한 것

비전공자가 가장 잘 헷갈리는 영역입니다. **같은 단어인데 구현 난이도가 천차만별**입니다.

| 기능 | 🟢 4일 안에 가능 | 🔴 4일 안에 불가능 |
|------|------------------|---------------------|
| **"친구랑 공유"** | URL을 카톡으로 보내기 (텍스트 복사) | 실시간 동시 편집 / 친구 그래프 |
| **"알림"** | 화면 안 토스트 메시지 | 카톡 푸시 / 시스템 푸시 알림 |
| **"결제"** | Stripe Payment Link 한 줄 붙이기 | 카드 정보 직접 받아 처리 / 정기결제 관리 |
| **"로그인"** | Google·카카오 OAuth (소셜 로그인) | 본인 비밀번호 시스템 + 비번 찾기 / 2FA |
| **"검색"** | 제목/이름 텍스트 매칭 | "비슷한 의미" 검색 (벡터 검색·임베딩) |
| **"추천"** | 최신순 / 인기순 정렬 | 개인 취향 기반 ML 추천 |
| **"지도"** | 카카오맵 위치 핀 1개 표시 | 실시간 길찾기 / 경로 최적화 |
| **"사진 업로드"** | 1장 업로드 후 표시 | OCR 텍스트 추출 / 얼굴 인식 |
| **"채팅"** | 게시글 + 댓글 (새로고침해서 봄) | 실시간 메시지 (타이핑 표시 등) |

> **핵심**: 같은 "공유"·"알림"이라도 **단순 버전**이 있고 **풀 버전**이 있습니다. 4일 안에는 무조건 단순 버전으로 갑니다.

## 3. 아이디어 단순화 — 큰 걸 4일짜리로 자르기 (5분)

본인 아이디어가 Red Flag에 걸려도 **포기하지 말고 잘라 보세요.** 다음 3가지 자르기 전략이 있습니다.

### 자르기 전략 1 — "사용자 1명 버전부터"

```
원래: 동네 헬스장 50명 회원 출석 체크 시스템
   → 4일짜리: 본인 1명용 운동 기록 일기

원래: 회사 직원 30명 휴가 신청·승인 시스템
   → 4일짜리: 본인 1명용 휴가 메모장
```

**다인용은 권한 분리·동기화로 복잡도가 폭발**합니다. 1인용으로 시작하면 같은 핵심 가치를 훨씬 단순하게 만들 수 있습니다.

### 자르기 전략 2 — "가장 자주 쓰는 1개 기능만"

```
원래: 가계부 (수입·지출 + 카드 자동 연동 + 예산 알림 + 그래프)
   → 4일짜리: 가계부 (수입·지출 한 줄 메모만)

원래: 식당 추천 (위치·가격·메뉴·후기·예약·결제)
   → 4일짜리: 즐겨찾기 식당 모음 (이름·메모만)
```

**부가 기능은 v2로 미루세요.** 핵심 1개만 잘 되면 v2에서 붙여도 됩니다.

### 자르기 전략 3 — "수동으로 풀 수 있는 건 수동으로"

```
원래: 모임 공지 자동 카톡 발송
   → 4일짜리: 공지 텍스트 자동 생성 + 사용자가 직접 카톡 복붙

원래: 마감일 자동 푸시 알림
   → 4일짜리: 화면에 "오늘 마감 3건" 배너 (앱 열어야 보임)

원래: 영수증 사진 자동 인식
   → 4일짜리: 영수증 한 줄 직접 입력
```

**자동화는 비싸고 어렵습니다.** 사용자 손 한 번 더 가는 단순 버전이 4일 안에 가능합니다.

### Before / After 단순화 예시 3개

```text filename="① 슬랙 클론 → 단순화"
원래 (4일 불가능):
- 팀 채팅 + 채널 + DM + 파일 공유 + 멘션 알림 + 검색

단순화 (4일 가능):
- 본인 일기 메모장 + "친구한테 공유" 버튼 (URL 카톡 복사)
- 실시간 X, 비동기 게시판 형태
```

```text filename="② 우버 클론 → 단순화"
원래 (4일 불가능):
- 실시간 위치 매칭 + 결제 + 평가 + 운전기사 관리

단순화 (4일 가능):
- 동네 카풀 게시판 (출발지·시간·인원만 게시)
- 매칭은 카톡으로 직접, 결제는 계좌이체
```

```text filename="③ 배민 클론 → 단순화"
원래 (4일 불가능):
- 식당 등록 + 주문 + 결제 + 배달 추적 + 리뷰

단순화 (4일 가능):
- 단골 식당 모음 (이름·메뉴·전화번호만)
- 클릭하면 전화 걸기 (배달 X, 결제 X)
```

## 4. 본인 아이디어 점검 체크리스트

다음 다 OK 면 PRD 단계로 진입할 준비 완료.

- [ ] 한 문장으로 앱을 정의할 수 있다 (예: "OOO한 사람을 위한 OOO 앱")
- [ ] Red Flag 8가지 중 해당하는 게 없다 (또는 단순화로 빠졌다)
- [ ] 핵심 기능이 **3개 이내**다
- [ ] 사용자가 1명 또는 1유형 위주다
- [ ] 화면이 머릿속에 4~6개 이내로 그려진다

## 자주 하는 실수

<Callout type="warning">
**실수 1: "이 정도는 별로 안 복잡할 거야"**
초보자에게 가장 흔합니다. **위 헷갈리는 함정 표를 한 번 더 보세요.** 같은 단어라도 단순 버전 / 풀 버전 차이가 큽니다.
</Callout>

<Callout type="warning">
**실수 2: 단순화하면서 본인 아이디어 가치를 다 잃었다고 느낌**
오해입니다. **핵심 가치는 보존하면서 부가만 빼는 게 단순화**입니다. "내 가족이 매일 약 먹는 시간을 안 까먹게" 라는 가치는 단순한 버전에서도 그대로 유지됩니다.
</Callout>

<Callout type="warning">
**실수 3: 복잡도 점검을 안 하고 PRD 작성**
복잡한 아이디어로 PRD를 잘 써도 Lovable이 다 못 만듭니다. 결국 다시 돌아와서 잘라야 합니다. **여기서 잘라두면 그 시간을 안 잃습니다.**
</Callout>
