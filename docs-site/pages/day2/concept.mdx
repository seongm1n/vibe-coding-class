# 내 앱 컨셉 정의서 작성

## 이 챕터의 목표

어제 가져온 본인 아이디어를 PRD로 변환합니다. 그리고 같은 챗봇에 PRD를 다시 던져서 **"이거 보고 뭐 만들지 상상되는가"** 자체 검증을 받습니다.

## 작성 순서

```
1. 어제 메모 꺼내기 (3줄)
   ↓
2. ChatGPT/Claude에 PRD 생성 프롬프트 던지기
   ↓
3. 답을 본인 상황에 맞게 직접 수정
   ↓
4. 같은 챗봇에 PRD를 다시 던져 자체 검증
   ↓
5. PRD 완성 (Lovable로 넘어갈 준비)
```

## 실습

### 실습 목표

본인 앱의 PRD 1장을 완성합니다.

### 준비물

- 어제 가져온 아이디어 메모 3줄
- ChatGPT 또는 Claude (Day 2는 Claude가 약간 더 좋습니다)

### 1단계 — 어제 메모를 한 단락으로 풀기

3줄짜리 메모를 1단락 정도의 글로 풀어 적습니다. 그래야 AI에게 던질 정보가 충분해집니다.

```text filename="예시 — 어제 메모를 한 단락으로"
나는 헬스장에 자주 다니지만, 운동 기록을 자꾸 까먹는다.
노트에 적으면 잃어버리고, 헬스장 앱은 너무 복잡해서 안 쓰게 된다.
30초 안에 운동 1개를 기록할 수 있는 단순한 앱이 있으면 좋겠다.
30대 직장인이고, 친구 1명과 운동을 같이 다닌다.
이번 주 기록을 친구한테 카톡으로 공유할 수 있으면 좋겠다.
```

### 2단계 — PRD 생성 프롬프트

위 단락을 넣어서 다음 프롬프트를 ChatGPT/Claude에 던집니다.

```text filename="PRD 생성 프롬프트"
다음은 내가 만들고 싶은 앱의 1단락 설명이다.

"[1단계에서 적은 한 단락]"

이 설명을 1페이지 PRD로 정리해 줘. 비개발자도 이해할 수 있게.

다음 형식으로:
1. 타겟 사용자 (한 명을 구체적으로 묘사)
2. 해결하려는 문제 (그 사람이 지금 겪는 불편을 구체적으로)
3. 핵심 기능 3개 (4일 안에 비전공자가 만들 수 있을 만큼 단순하게)
4. 가장 중요한 화면 2~3개의 구성 요소
   - 메인 화면에 무엇이 보이는가
   - 사용자가 가장 자주 누르는 버튼은 무엇인가
5. 디자인 톤 (색상·분위기 한 문장)
6. 모바일/데스크톱 우선

추가 조건:
- 노코드 도구(Lovable)로 만들 거라 너무 복잡한 기능은 빼 줘
- 백엔드(데이터 저장)는 다음 날에 추가할 거니까, 오늘은 화면 위주로
- 한국어로
```

### 3단계 — 답을 직접 수정

AI가 만든 PRD에는 본인이 의도하지 않은 부분이 섞여 있을 수 있습니다. **그대로 쓰지 말고**, 다음 항목을 한 번씩 짚으면서 본인 말로 다시 적으세요.

- [ ] 타깃 사용자 묘사가 진짜 본인이 알고 있는 사람과 비슷한가?
- [ ] 문제 부분이 본인이 진짜 느낀 불편인가?
- [ ] 핵심 기능 3개가 정말 다 필요한가? 1개는 빼도 되지 않을까?
- [ ] 화면 구성이 머릿속에 그려지는가?

본인 색깔이 들어가야 좋은 PRD입니다.

### 4단계 — AI 자체 검증

손질이 끝난 PRD를 같은 챗봇에 다시 던지세요. 본인이 못 보는 모호한 부분을 AI가 짚어 줍니다.

```text filename="PRD 자체 검증 프롬프트"
다음은 내가 작성한 PRD다.

"[방금 손질한 PRD 그대로 붙여넣기]"

이 PRD를 보고 다음을 알려줘:
1. 이걸로 화면을 만든다면 어떤 앱이 나올지 한 문장 요약
2. 화면 만들 때 헷갈릴 만한 모호한 부분 3개
3. 빠진 정보가 있다면 무엇인지
4. 너무 복잡해서 4일 안에 못 끝낼 것 같은 기능이 있다면 그게 뭔지

비개발자도 이해할 수 있게 답해줘.
```

AI 답이 본인 의도와 다르면 PRD가 모호한 것입니다. 짚어 준 부분을 보강해서 다시 던지세요. **이걸 1~2번 반복하면 PRD가 눈에 띄게 좋아집니다.**

<Callout type="info">
**AI 검증의 본질**: 본인이 머릿속에만 갖고 있는 정보는 PRD 글자에는 없습니다. AI는 글자만 보고 판단하니, AI가 헷갈리면 Lovable도 똑같이 헷갈립니다. **AI에게 PRD를 검증시키는 게 가장 빠른 자체 점검 방법**입니다.
</Callout>

### 5단계 — Lovable용 프롬프트로 변환

PRD가 완성됐으면, 이걸 Lovable에 던질 1개 프롬프트로 변환합니다. 다음 템플릿을 채워 보세요.

```text filename="Lovable용 첫 프롬프트 템플릿"
[앱 이름] 만들어 줘.

타겟 사용자: [한 줄 묘사]
이 사용자가 겪는 문제: [한 줄]
앱이 해결하는 방식: [한 줄]

[메인 화면]
- 화면 상단: [무엇이 보이는가]
- 화면 중앙: [무엇이 보이는가]
- 화면 하단: [무엇이 보이는가]
- 가장 큰 버튼: [어떤 동작인가]

[서브 화면]
- 무엇을 보여 주는 화면인가: [설명]
- 메인 화면에서 어떻게 들어오는가: [설명]

스타일:
- 미니멀, 깔끔, 둥근 모서리
- 색상: [본인 컬러]
- 폰트: 한국어가 잘 보이는 산세리프 (Pretendard 또는 시스템 폰트)
- 모바일 우선 (가로 360~400px), 데스크톱에서도 잘 보이게

데이터는 일단 가짜 데이터(샘플 5개)로 채워 줘.
백엔드 연결은 다음 단계에서 추가할 거야.

언어: 한국어
```

## 막힌 사람용 백업 PRD 5개

본인 아이디어가 도저히 안 떠오르거나 시간이 너무 흐른다면, 아래 PRD 중 하나를 그대로 가져다 쓰세요. **그대로 써도 부끄러워할 필요 없습니다.** 강의 목표는 "워크플로우를 손에 익히는 것"이지 "독창적인 아이디어 생산"이 아닙니다.

### 1) 동네 식당 영업시간 모음

```text filename="백업 PRD 1"
세종시 어진동 직장인을 위한 점심 식당 모음 앱.

타겟: 정부청사 근처 직장인 (점심 12~13시)
문제: 매번 같은 곳만 가게 되고, 새 곳을 찾기 번거로움
핵심 기능 3개:
1) 오늘의 추천 식당 3곳 (거리·메뉴·가격대 표시)
2) 거리/메뉴/가격대 필터
3) 즐겨찾기 별표

화면:
- 메인: 추천 카드 3개 + 필터 버튼 + 즐겨찾기 탭
- 상세: 식당 정보 + 영업시간 + 별점

디자인: 따뜻한 주황 + 흰 배경, 둥근 모서리
모바일 우선
```

### 2) 운동 기록 일기

```text filename="백업 PRD 2"
직장인 헬스 기록용 단순 일기 앱.

타겟: 헬스장 주 3회 다니는 30대 직장인
문제: 운동 기록을 자꾸 까먹음. 헬스장 앱은 너무 복잡함
핵심 기능 3개:
1) 운동 1개를 30초 안에 기록 (종목/무게/횟수)
2) 이번 주 기록 한눈에 보기 (요일별 카드)
3) 친구 1명에게 이번 주 요약 공유 (텍스트 형태)

화면:
- 메인: 오늘의 기록 입력 폼 (드롭다운 + 숫자 2개 + 저장 버튼)
- 기록 보기: 이번 주 7개 카드
- 공유 화면: 이번 주 요약 + 복사 버튼

디자인: 미니멀, 검은 글자, 짙은 주황 강조
모바일 우선
```

### 3) 가계부 (수입·지출 메모)

```text filename="백업 PRD 3"
프리랜서를 위한 단순 영수증 메모 앱.

타겟: 프리랜서 디자이너, 매월 수입·지출 정리에 시간 쓰는 사람
문제: 엑셀 가계부는 매번 열기 귀찮음. 영수증을 그때그때 적고 싶음
핵심 기능 3개:
1) 한 줄 메모로 수입/지출 기록 (날짜·금액·카테고리·메모)
2) 이번 달 합계 (수입 / 지출 / 차액)
3) 카테고리별 합계 보기

화면:
- 메인: 이번 달 합계 + 빠른 입력 폼
- 목록: 이번 달 거래 내역 (날짜순)
- 카테고리: 카테고리별 합계 막대 그래프

디자인: 흰 배경 + 진한 회색 글자, 강조색 1개
모바일 우선
```

### 4) 독서 메모

```text filename="백업 PRD 4"
책 좋아하는 사람용 가벼운 독서 일기.

타겟: 한 달에 책 1~2권 읽는 사람, 다 읽고 나면 어떤 내용이었는지 기억이 흐려짐
문제: 메모 앱들은 너무 자유로워서 결국 안 쓰게 됨
핵심 기능 3개:
1) 책 1권을 한 줄 평 + 별점 + 인상 깊은 구절 1개로 기록
2) 읽은 책 목록 (커버 이미지 그리드)
3) 책 검색 (제목/저자)

화면:
- 메인: 읽은 책 그리드 + "+책 추가" 버튼
- 추가: 제목/저자/별점/한 줄 평/구절 입력 폼
- 상세: 책 정보 + 본인 메모

디자인: 책 표지가 잘 보이는 그리드, 베이지/크림 색
모바일·태블릿 둘 다
```

### 5) 공구 모임 정리

```text filename="백업 PRD 5"
디저트·커피 좋아하는 사람들의 공구 정리 앱.

타겟: 커뮤니티에서 공동구매를 자주 하지만 정리가 안 되는 사람
문제: 카톡방에서 누가 몇 개 시켰는지 추적이 어려움
핵심 기능 3개:
1) 공구 1건 등록 (상품명/단가/모집 인원/마감일)
2) 참여 신청 (이름 + 수량 입력)
3) 마감 후 정산표 자동 생성 (참여자별 금액)

화면:
- 메인: 진행 중 공구 목록 카드 (마감일 임박 표시)
- 상세: 상품 정보 + 참여자 명단 + 신청 폼
- 정산: 참여자별 결제 금액 + 입금 계좌

디자인: 따뜻한 파스텔 + 손으로 쓴 듯한 폰트
모바일 우선
```

## 체크포인트
