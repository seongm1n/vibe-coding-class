# Lovable로 첫 화면 만들기

## 이 챕터의 목표

앞에서 만든 PRD를 Lovable에 던져 **본인 앱의 첫 화면**을 띄웁니다. 가능하면 서브 화면 1개도 추가합니다.

## 시작 전 — Lovable 무료 한도 미리 알기

본격적으로 메시지를 던지기 전에 **본인 한도부터** 알고 시작하세요. 한도가 빨리 떨어지는 게 Day 2~4에 가장 큰 진도 방해 요인입니다.

| 한도 항목 | Free 플랜 |
| --- | --- |
| **메시지(채팅)** | **하루 5개 / 월 30개** |
| **Cloud(백엔드)** 사용량 | 월 $25 무료 한도 (Day 3부터 의미 있음) |
| **AI 기능** 사용량 | 월 $1 무료 한도 |
| 배포 서브도메인 | 5개 |

소비 규칙:

- **Plan 모드**: 메시지 1번 = 크레딧 1개
- **Agent 모드**: 작업 복잡도에 따라 가변 (단순 작업 1개, 복잡한 멀티스텝은 더 많이)
- **"Try to fix" 버튼은 무료** (크레딧 안 깎임)
- **"이전 버전으로 되돌리기"도 무료**

<Callout type="warning">
**하루 5개라는 게 핵심입니다.** 그래서 이 강의의 모든 프롬프트는 "한 번에 모든 요구사항을 묶어서" 던지는 식으로 짜여 있습니다. **메시지를 쪼개면 그날 진도가 끊깁니다.**
</Callout>

<Callout type="info">
**한시 프로모션 안내**: 무료 플랜의 월 $25 Cloud + $1 AI 한도는 **2026년 5월까지** 한시 적용입니다. 강의 후 계속 쓸 계획이라면 [공식 가격 페이지](https://lovable.dev/pricing)에서 그 시점 정책을 다시 확인하세요. 정확한 표는 Day 3 [Lovable Cloud로 백엔드 켜기](/day3/lovable-cloud)에 있습니다.
</Callout>

## Lovable 화면 구성

[Lovable](https://lovable.dev)에 로그인하면 다음 영역이 보입니다.

```
┌─────────────────────────────────────────────────┐
│  채팅창 (왼쪽 또는 아래)  │  미리보기 (오른쪽 또는 위) │
│  - 프롬프트 입력          │  - 만든 화면이 실시간으로  │
│  - AI 답변                │    보이는 곳              │
└─────────────────────────────────────────────────┘
```

- **채팅창**: 여기에 한국어로 시키면 됩니다.
- **미리보기**: 화면이 자동으로 그려집니다. 모바일/데스크톱 전환 가능합니다.
- **메시지 카운터**: 화면 어딘가에 "오늘 남은 메시지 X개"가 표시됩니다. 항상 확인하세요.

## 실습

### 실습 목표

본인 PRD로 메인 화면 1개 + 서브 화면 1개를 만듭니다.

### 준비물

- Lovable 로그인된 상태
- 앞에서 준비한 Lovable용 첫 프롬프트
- (선택) Day 1에 만든 로고 PNG

### 단계 1 — 새 프로젝트 생성

1. Lovable 메인 → **"New Project"** 또는 가운데 입력창
2. 프로젝트 이름 입력 (예: `fitcheck`, `lunchpicker` 등 영어 권장)
3. 첫 프롬프트 붙여넣기

<Callout type="info">
**프로젝트 이름은 짧고 영어로** 짓는 게 좋습니다. 나중에 도메인이나 URL로 쓰일 수 있기 때문입니다.
</Callout>

### 단계 2 — 첫 프롬프트 던지기

준비한 프롬프트를 그대로 붙여 넣고 Enter. AI가 1~3분 안에 첫 화면을 만들어 줍니다.

```text filename="첫 프롬프트 예시 (운동 기록 앱 기준)"
FitCheck 운동 기록 일기 앱 만들어 줘.

타겟: 헬스장 주 3회 다니는 30대 직장인
문제: 운동 기록을 자꾸 까먹음, 헬스장 앱은 너무 복잡
앱이 해결하는 방식: 30초 안에 운동 1개를 기록할 수 있게

[메인 화면]
- 화면 상단: 앱 이름과 오늘 날짜
- 화면 중앙: 운동 입력 폼 (종목 드롭다운, 무게 입력, 횟수 입력, 메모란)
- 화면 하단: 큰 "기록 저장" 버튼
- 입력 폼 위에 작은 글씨로 "이번 주 운동 X회" 통계

[서브 화면 — 이번 주 기록]
- 메인 화면 우측 상단의 "이번 주" 버튼으로 진입
- 요일별 카드 7개, 각 카드에 그날 운동 종목과 횟수
- 비어 있는 요일은 "쉼" 표시

스타일:
- 미니멀, 깔끔, 둥근 모서리 (border-radius: 12px 정도)
- 색상: 흰 배경 + 짙은 주황(#c8401a) 강조
- 폰트: Pretendard 또는 한국어 잘 보이는 시스템 폰트
- 모바일 우선 (가로 360~400px), 데스크톱에서도 가운데 정렬로 잘 보이게

데이터는 일단 가짜 샘플 5개로 채워 줘.
백엔드는 다음 단계에서 추가할 거야.

언어: 한국어
```

### 단계 3 — 결과 보기 + 미세 수정

화면이 나오면 미리보기에서 다음을 체크하세요.

- [ ] 색상이 의도한 대로인가
- [ ] 글자 크기가 너무 작지 않은가
- [ ] 모바일 미리보기에서도 화면이 깨지지 않는가
- [ ] 버튼이 적절한 위치에 있는가
- [ ] 한국어가 잘 보이는가

마음에 안 드는 부분을 **한 메시지에 묶어서** 수정 요청합니다.

```text filename="수정 프롬프트 예시"
다음을 한 번에 수정해 줘:
- "기록 저장" 버튼을 화면 하단 고정으로 (스크롤해도 항상 보이게)
- 메인 화면 입력 폼의 라벨을 더 크고 진하게
- 색상 강조를 #c8401a 대신 살짝 더 어두운 #b03515로
- 모바일에서 폼 사이 간격을 16px로 늘려 줘
```

### 단계 4 — 서브 화면이 잘 안 만들어졌으면

서브 화면이 빠졌거나 이상하면 다음 메시지로 추가합니다.

```text filename="서브 화면 추가 프롬프트"
"이번 주 기록" 화면을 다음대로 만들어 줘:
- 상단에 "이번 주 운동" 제목 + 메인 화면으로 돌아가는 ← 버튼
- 요일별 카드 7개를 세로로 나열
  - 카드 안에는 요일(월~일), 그날 운동 종목, 무게/횟수
  - 운동 안 한 요일은 회색 배경에 "쉼"
- 화면 하단에 "이번 주 총 X회 운동" 통계 카드

샘플 데이터로 일/수/금에 운동 기록 채워 줘.
```

### 단계 5 — 본인 로고 적용 (선택)

Day 1에 만든 로고를 적용합니다. Lovable에 이미지를 직접 업로드할 수도 있고, 텍스트로 시킬 수도 있습니다.

```text filename="로고 적용 프롬프트"
앱 상단의 "FitCheck" 텍스트 옆에 로고 이미지를 넣어 줘.
로고 파일을 첨부할게. 크기는 32px, 텍스트 왼쪽에 정렬.
```

## 메시지 아껴쓰기 황금률

<Callout type="warning">
**Lovable 무료 한도는 하루 5개 / 월 30개입니다.** 메시지 1개로 최대한 많이 시키세요. 위 "시작 전 — Lovable 무료 한도" 섹션 참고.
</Callout>

| 나쁜 패턴 | 좋은 패턴 |
| --- | --- |
| "색을 바꿔 줘" → 결과 → "이번엔 글자 크기" → 결과 ... (5번에 끝남) | "색·글자 크기·간격·버튼 위치를 다음대로 한 번에 바꿔 줘: ..." (1번에 끝남) |
| 결과 보고 마음에 안 들면 새 프로젝트 생성 | 같은 프로젝트 안에서 "이전 버전으로 되돌리기" |
| 추상적으로 "예쁘게 만들어 줘" | "둥근 모서리 12px, 그림자 살짝, 간격 16px로" 구체적으로 |

## 진도가 빠른 사람은

서브 화면 1개를 더 추가하거나, **상세 페이지**나 **입력 폼**을 따로 만들어 보세요.

```text filename="진도 빠른 사람용 추가 프롬프트"
"운동 종목 추가" 화면을 만들어 줘:
- 사용자가 새로운 운동 종목을 만들 수 있게
- 입력: 종목 이름, 부위(가슴/등/하체/어깨/팔/유산소), 단위(kg/회/분)
- 저장하면 메인 화면 드롭다운에 새 종목이 나타나야 함
```

## 진도가 느린 사람은

PRD가 너무 복잡한 게 원인일 수 있습니다. 다음을 시도하세요.

1. **백업 PRD로 갈아타기** — [내 앱 컨셉 정의서](/day2/concept) 마지막의 백업 PRD 5개 사용
2. **기능 1개로 줄이기** — 핵심 기능 3개 중 가장 중요한 1개만 만들기
3. **샘플 데이터 늘리기** — "샘플 데이터를 10개로 늘려서 화면이 풍성하게 보이게"

## Lovable이 막혔을 때 — 백업 도구

무료 한도 소진, 서버 응답 없음, 결과가 너무 이상함 등의 경우:

### v0 (Vercel)

1. [v0.dev](https://v0.dev) 접속
2. 같은 PRD 프롬프트를 그대로 붙여넣기
3. UI 위주 결과 → 화면 1개를 받아서 본인 머릿속 앱과 비교

### Bolt (StackBlitz)

1. [bolt.new](https://bolt.new) 접속
2. 프롬프트 붙여넣기 → 바로 풀 프로젝트 생성
3. 더 강력하지만 결과가 복잡할 수 있음

<Callout type="info">
**도구를 갈아탈 때 PRD는 그대로 사용하세요.** 도구가 달라도 PRD가 잘 만들어졌다면 비슷한 결과가 나옵니다. 시간 낭비를 막는 핵심 비결입니다.
</Callout>

## 자주 하는 실수

<Callout type="error">
**실수 1: "내 화면이 이상하게 나와요"**
대부분 PRD가 모호한 게 원인입니다. PRD를 다시 보고 빠진 부분을 찾으세요. (특히 "화면 구성" 부분)
</Callout>

<Callout type="error">
**실수 2: 메시지를 다 써 버린 뒤 "내일이 또 있겠지"**
무료 한도가 소진되면 그날은 끝입니다. **메시지를 써야 할 때를 신중하게** 고르세요. 결과를 충분히 본 뒤 다음 메시지를 보내세요.
</Callout>

<Callout type="error">
**실수 3: 결과 화면을 보고 머릿속에 있는 기준과 비교**
"내가 생각한 거랑 다른데?" → PRD에 그 기준이 적혀 있지 않은 경우가 대부분입니다. 머릿속 기준을 PRD에 적어 다시 시키세요.
</Callout>

## 체크포인트
