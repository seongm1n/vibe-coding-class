# Lovable 반복 개선 — 풀 사용자 흐름 만들기

## 이 챕터의 목표

앞 챕터에서 **메인 화면 + 서브 화면 1개**를 만들었습니다. 그건 시작입니다. 이번 챕터에서는 **사용자가 처음 들어와서 한 사이클을 끝낼 수 있는 흐름**을 만듭니다.

```
지금까지            이번 챕터 후
─────────           ─────────────
화면 2개            화면 4~6개
"화면이 떴다"        "사용자 흐름이 완성됐다"
샘플 데이터 5개      데이터 20개 + 진짜 같은 콘텐츠
정상 상태만          빈 / 로딩 / 에러 상태 모두
모니터에서만 봄      본인 폰에서 직접 만져 봄
```

> **이 챕터가 Day 2의 진짜 핵심**입니다. 첫 화면은 출발점이고, **반복 개선을 거쳐야 진짜 앱처럼 보입니다.**

## 1. 사용자 흐름 그리기 (5분)

코딩 시작 전에 종이·메모장에 흐름을 한 번 그려 봅니다. 머릿속에만 있는 흐름은 화면으로 옮기기 어렵습니다.

```text filename="운동 기록 앱 — 흐름 예시"
  [홈 (목록)]
    │ "+추가" 버튼
    ↓
  [운동 입력]
    │ 저장
    ↓
  [홈 (목록 갱신됨)]
    │ 카드 클릭
    ↓
  [운동 상세]
    │ "수정"           "삭제"
    ↓                  ↓
  [수정 입력]         [삭제 확인 모달]
    │ 저장              │ 삭제 확인
    ↓                  ↓
  [상세 (변경 반영)]   [홈 (목록에서 사라짐)]
```

본인 앱의 흐름을 4~6단계로 그립니다. **이게 곧 만들어야 할 화면 목록**입니다.

## 2. 빠진 화면 한 메시지로 묶어서 만들기 (20분)

앞에서 만든 화면 외에 빠진 화면을 **한 메시지에 묶어서 한 번에** 시킵니다. Lovable 메시지 절약 + 일관성 유지를 위해.

```text filename="여러 화면 한 번에 추가하는 프롬프트"
다음 화면들을 한 번에 추가해 줘. 모든 화면은 메인 화면과 동일한 색상·폰트 톤으로.

[1) 운동 입력 화면]
- 진입 경로: 메인 화면 우측 상단 "+추가" 버튼
- 입력 항목: 종목(드롭다운) / 무게(kg) / 횟수 / 날짜 / 메모
- 저장 버튼: 화면 하단 고정 (스크롤해도 항상 보이게)
- 취소: 좌측 상단 "←" 화살표

[2) 운동 상세 화면]
- 진입 경로: 메인 목록의 카드 클릭
- 표시 정보: 입력했던 모든 정보 + 등록 일시
- 우측 상단: "수정" 버튼, "삭제" 버튼

[3) 수정 화면]
- 진입 경로: 상세 화면의 "수정" 버튼
- 입력 화면과 동일하지만 기존 값이 미리 채워져 있음
- 저장 시 상세 화면으로 돌아가며 변경 내용 반영

[4) 삭제 확인 모달]
- 진입 경로: 상세 화면의 "삭제" 버튼
- "정말 삭제하시겠어요?" + 취소 / 삭제 버튼
- 삭제 누르면 메인 목록으로 돌아가며 해당 카드 사라짐

샘플 데이터로 입력 화면에 운동 1개 미리 채워서 보여 줘.
모바일 우선.
```

3~4개 화면이 한 메시지로 만들어집니다. **메시지 1개로 4화면이면 효율 매우 좋음.**

## 3. PRD 재투입 — 결과 보고 다시 던지기 (15분)

> **반복 개선의 핵심 사이클**:
> 화면 → 본인 눈으로 점검 → PRD에 빠진 부분 추가 → 다시 던지기

첫 결과를 보면, 본인이 처음 PRD에 적지 않은 **암묵적인 기준**들이 떠오릅니다. 이게 곧 PRD v2 의 재료입니다.

### 떠오르는 생각 → PRD 한 줄로 변환

| 떠오르는 생각 | PRD에 추가할 한 줄 |
|---------------|---------------------|
| "버튼 색이 너무 튀네" | 버튼 색은 메인 톤보다 5~10% 어두운 색으로 |
| "글씨가 너무 작아" | 본문 14px → 16px로 (어르신 사용자 가정) |
| "카드 사이가 너무 붙었네" | 카드 간격 16px 이상 |
| "한국어가 어색해" | 카피는 자연스럽게, 번역 투 금지 |
| "input에 안내 글이 없어" | 모든 입력 필드에 placeholder 한국어로 |
| "저장 버튼이 눌렸는지 모르겠음" | 버튼은 눌리기 전/후 색이 살짝 다르게 (시각적 피드백) |

이렇게 5~10개를 모은 뒤 **한 메시지에 묶어서 다시 시킵니다.**

```text filename="PRD v2 재투입 프롬프트"
앞에서 만든 PRD에 다음 기준을 추가하기로 결정했어:

- 버튼 색은 메인 톤보다 5~10% 어두운 색으로
- 본문 14px → 16px로 (어르신 사용자 가정)
- 카드 간격 16px 이상
- 한국어 카피는 자연스럽게, 번역 투 금지
- 모든 입력 필드에 한국어 placeholder
- 저장 버튼은 누르기 전/후 시각적 피드백
- 모든 버튼 최소 높이 44px (모바일 터치 영역)

위 기준으로 **모든 화면을 한 번에 점검해서 수정**해 줘.
```

이걸 **1~2번 더 돌리면** 화면 품질이 눈에 띄게 올라갑니다. 이게 진짜 강력한 부분입니다.

## 4. 콘텐츠 풍성하게 — Day 1 도구 재활용 (15분)

화면이 비어 보이면 "진짜 앱" 같은 느낌이 안 납니다. **Day 1에 만진 AI 도구들을 여기서 재활용**합니다.

### 4-1. 샘플 데이터 늘리기

```text filename="더미 데이터 풍성하게 프롬프트"
샘플 데이터를 5개에서 20개로 늘려 줘.

- 운동 종목: 벤치프레스, 데드리프트, 스쿼트, 풀업, 푸시업, 런닝, 사이클, 요가, 플랭크 등 다양하게
- 무게: 0kg(맨몸 운동) ~ 100kg 사이 현실적인 값
- 횟수: 종목별로 자연스러운 범위 (런닝은 분 단위, 웨이트는 횟수)
- 날짜: 최근 4주에 골고루 분포 (매일 운동한 척 X, 주 3~4회 정도)
- 메모: 절반은 비워두고, 절반은 짧은 한 줄 메모
```

### 4-2. AI 이미지로 화면 채우기

```text filename="이미지 자동 채우기"
운동 카드의 썸네일 이미지를 Unsplash의 fitness 카테고리에서 자동으로 가져와 줘.
종목별로 매칭되는 이미지로 (예: 벤치프레스 → 벤치프레스 동작 사진, 런닝 → 러닝 사진).
이미지는 카드 좌측에 80x80px 정사각형 + 둥근 모서리 8px.
```

### 4-3. 한국어 카피 다듬기 (ChatGPT 옆 탭)

화면에 들어간 영어·번역 투 표현은 별도 탭의 ChatGPT에 던져서 자연스럽게 만든 뒤 Lovable에 다시 입력합니다.

```text filename="ChatGPT용 한국어 카피 다듬기"
다음은 운동 기록 앱 화면에 들어가는 텍스트들이다.
번역 투를 한국어 자연스러운 표현으로 바꿔 줘. 길이는 비슷하게 유지.

- "운동을 기록하세요"
- "이번 주 너의 진행"
- "운동 데이터를 보내려고 합니다"
- "더보기"
- "데이터가 없습니다"
- "저장 중입니다"
```

ChatGPT가 다듬은 결과를 Lovable에 한 메시지로 묶어서 적용:

```text filename="Lovable에 카피 적용"
다음 텍스트들을 한꺼번에 교체해 줘:
- "운동을 기록하세요" → "오늘 한 운동 적어두기"
- "이번 주 너의 진행" → "이번 주 운동 흐름"
- "더보기" → "더 보기"
... (이런 식으로)
```

## 5. 인터랙션 상태 — 빈 / 로딩 / 에러 (15분)

진짜 앱이라면 **정상 상태 외에도 다양한 상황**이 있습니다. 비전공자가 가장 빠뜨리는 영역인데, **이게 있냐 없냐가 "장난감 vs 진짜 앱"의 차이**입니다.

### 5-1. 빈 상태 (Empty State)

데이터가 하나도 없을 때 화면이 어떻게 보이는가.

```text filename="빈 상태 추가 프롬프트"
운동 목록 화면에 기록이 하나도 없을 때 다음과 같이 보여 줘:

- 화면 가운데에 친근한 일러스트 1개 (Lovable이 자동 생성)
- 제목: "아직 첫 기록이 없어요"
- 부제: "오른쪽 위 + 버튼으로 첫 운동을 기록해 보세요"
- 큰 "지금 기록하기" 버튼 (메인 톤 색)

핵심: 사용자가 **다음에 무엇을 해야 할지 명확히 보이게**.
```

### 5-2. 로딩 상태

```text filename="로딩 상태 추가 프롬프트"
다음 두 가지 로딩 상태를 추가해 줘:

[저장 버튼을 눌렀을 때]
- 버튼이 비활성화되고
- "저장 중..." 텍스트 + 작은 스피너로 변경
- 1초 후 "저장됨 ✓" 으로 짧게 변경됐다가 원래 모습으로

[목록 화면 진입 시]
- 카드들이 회색 placeholder(skeleton)로 먼저 보임
- 0.5초 후 진짜 데이터로 채워지는 애니메이션
```

### 5-3. 에러 상태

```text filename="에러 상태 추가 프롬프트"
저장이 실패했을 때:
- 화면 하단에 빨간색 토스트 메시지
  "저장에 실패했어요. 잠시 후 다시 시도해 주세요."
- 토스트는 3초 후 자동으로 사라짐
- **입력했던 내용은 사라지지 않게** — 사용자가 다시 저장 누르기만 하면 되게

네트워크가 끊겼을 때:
- 화면 상단에 회색 배너
  "오프라인 상태입니다. 인터넷 연결을 확인해 주세요."
```

<Callout type="info">
**왜 이 3가지가 중요한가?**
정상 상태만 있는 앱은 "데모 영상에는 멋있지만 실제로 쓰면 답답한 앱"이 됩니다. 빈/로딩/에러 3가지를 챙기면 사용자가 **무슨 일이 일어났는지 항상 이해할 수 있게** 됩니다.
</Callout>

## 6. 본인 폰에서 미리보기 (5분)

Lovable은 **공유 URL**을 자동으로 만들어 줍니다.

### 단계

1. Lovable 화면 우측 상단 → **"Share"** 또는 **"Preview"** 버튼
2. 공유 URL이 나옴 (예: `https://your-app.lovable.app`)
3. 본인 폰 카메라로 화면의 QR 찍기 — 또는 — URL을 본인에게 카톡으로 보내기
4. 폰에서 열기 → **본인이 만든 앱이 본인 폰에서 굴러감**

<Callout type="info">
**비전공자에게 가장 짜릿한 순간**입니다. "내가 만든 앱이 진짜 내 폰에 떴다"는 체감은 **다른 어떤 학습보다 강력**합니다. 강사도 강의 중 이 순간을 못 넘기게 시간 배분하세요.
</Callout>

### 폰에서만 보이는 5가지 점검

모니터 미리보기에서는 안 보이는 것들이 폰에서 드러납니다.

- [ ] 글자가 너무 작지 않은가
- [ ] 버튼이 손가락으로 누르기 편한가 (44px 이상)
- [ ] 가로 스크롤이 발생하지 않는가
- [ ] 키보드가 입력 폼을 가리지 않는가
- [ ] 화면 회전(가로/세로) 시 깨지지 않는가

문제 있으면 메모해 두고 다음 PRD v3 사이클로 반영합니다.

## 7. 처음부터 끝까지 한 사이클 (5분)

만든 화면들을 **사용자 시점으로 한 사이클** 돌려 봅니다.

```
1. 앱 열기 → 메인 화면 잘 보이는가
2. "+추가" 누르기 → 입력 화면 잘 이동하는가
3. 입력 → 저장 → 목록으로 자연스럽게 돌아오는가, 새 카드가 보이는가
4. 카드 클릭 → 상세 화면 잘 뜨는가
5. 수정 → 저장 → 변경 내용이 반영되어 상세에 보이는가
6. 삭제 → 확인 모달 → 삭제 후 목록에서 사라지는가
```

막히는 곳이 있으면 그게 곧 **다음 메시지로 고칠 부분**입니다.

## 본인 도메인이 운동이 아니라면? — 4가지 흐름 예시

이 챕터의 예시는 운동 기록 앱 기준이지만, **백업 PRD 5개**와 매칭되는 흐름 예시를 모두 정리했습니다. 본인 앱 도메인이 다르다면 아래에서 가까운 걸 골라 응용하세요.

### A. 식당 추천 / 모음

```text filename="식당 추천 흐름"
[홈 — 오늘 추천 3곳]
  │ 카드 클릭         "필터" 버튼
  ↓                   ↓
[식당 상세]          [필터 화면 — 거리/메뉴/가격]
  │ "★ 즐겨찾기"        │ 적용
  ↓                   ↓
[홈 (★ 표시 반영)]    [홈 (필터링됨)]

[추가 화면]
- 즐겨찾기 전용 목록 화면
- 검색 화면 (식당 이름)

[빈/로딩/에러]
- 빈: "주변 식당 정보가 없어요. 위치 권한을 확인해 주세요"
- 로딩: 카드 skeleton 3개
- 에러: 위치 권한 거부 시 안내 + "권한 다시 요청" 버튼
```

### B. 가계부 (수입·지출)

```text filename="가계부 흐름"
[홈 — 이번 달 합계 + 거래 목록]
  │ "+" 버튼
  ↓
[거래 입력 — 수입/지출 토글, 금액, 카테고리, 메모]
  │ 저장
  ↓
[홈 (목록 갱신, 합계 갱신)]
  │ 거래 카드 클릭
  ↓
[거래 상세] → 수정 / 삭제

[추가 화면]
- 카테고리별 합계 (월별 막대 그래프)
- 기간 검색 (예: "이번 주만 보기")

[빈/로딩/에러]
- 빈: "이번 달 첫 거래를 기록해 보세요"
- 로딩: 카드 skeleton + 합계 자리에 "—원"
- 에러: "잠시 후 다시 시도해 주세요"
```

### C. 독서 메모

```text filename="독서 메모 흐름"
[홈 — 읽은 책 그리드 (표지)]
  │ "+ 책 추가"
  ↓
[책 추가 — 제목, 저자, 별점, 한 줄 평, 인상 깊은 구절]
  │ 저장
  ↓
[홈 (그리드에 새 표지 추가)]
  │ 책 클릭
  ↓
[책 상세 — 별점·평·구절 모두 표시] → 수정 / 삭제

[추가 화면]
- 검색 (제목/저자)
- 별점·연도 필터

[빈/로딩/에러]
- 빈: "첫 책을 기록해 볼까요?" + 책 일러스트
- 로딩: 그리드 skeleton (회색 정사각형 6개)
- 에러: 책 표지 이미지 로드 실패 시 회색 placeholder
```

### D. 공구 모임 정리

```text filename="공구 흐름"
[홈 — 진행 중 공구 카드 (마감일 임박 표시)]
  │ "+ 공구 등록"
  ↓
[공구 등록 — 상품명, 단가, 모집 인원, 마감일]
  │ 등록
  ↓
[홈 (새 카드 추가)]
  │ 카드 클릭
  ↓
[공구 상세 + 참여자 명단]
  │ "참여 신청" 버튼
  ↓
[참여 신청 — 이름, 수량] → 저장 → 명단 갱신

[마감 후 자동 진입]
[정산표 — 참여자별 결제 금액 + 입금 계좌]

[추가 화면]
- 마감 임박 알림 화면
- 본인 참여 이력

[빈/로딩/에러]
- 빈: "진행 중인 공구가 없어요. 첫 공구를 등록해 보세요"
- 로딩: 카드 skeleton + 마감일 자리 점선
- 에러: 마감일 지난 공구 클릭 시 "이미 마감된 공구예요"
```

### E. 운동 기록 (이 챕터 본문 예시)

본문 위 예시 참고. (홈 → 입력 → 상세 → 수정/삭제 흐름)

## 자주 하는 실수

<Callout type="error">
**실수 1: 화면을 한 번에 한 개씩만 만들기**
한 메시지로 3~4 화면 묶을 수 있는데 한 화면씩 만들면 메시지 한도가 빨리 소진됩니다. **묶어 시키는 습관**을 들이세요.
</Callout>

<Callout type="error">
**실수 2: 빈 상태 / 에러 상태를 빼먹음**
정상 상태만 만들면 "데모용 가짜 앱"처럼 보입니다. **빈 상태 1개라도 꼭** 추가하세요.
</Callout>

<Callout type="error">
**실수 3: 폰에서 안 보고 모니터에서만 만족함**
모니터 미리보기에서는 멀쩡해도 폰에서 글자가 작거나 버튼을 못 누르는 경우 흔합니다. **꼭 본인 폰에서 한 번 열어 보세요.**
</Callout>

<Callout type="error">
**실수 4: PRD를 처음 한 번만 던지고 끝**
PRD는 **반복 개선의 도구**입니다. 첫 결과 보고 PRD에 추가할 줄이 안 떠오르면 그건 PRD를 "고정된 문서"로 보고 있는 것. 화면 보고 떠오르는 모든 잔소리를 PRD에 한 줄씩 추가하세요.
</Callout>

## 진도가 막힌 사람 — 안전망

여기서 막힌다고 강의가 망한 게 아닙니다. 다음 중 하나를 시도하세요.

### 1) Lovable 메시지가 다 떨어졌을 때

| 증상 | 대응 |
|------|------|
| "오늘 메시지 한도 도달" 메시지 | **v0** 또는 **Bolt**로 전환 (PRD 그대로 재사용) |
| 결과가 너무 이상함 | 같은 PRD로 **새 프로젝트 다시 생성**하면 깨끗한 결과 |
| 응답이 안 옴 (서버 부하) | 5분 기다리거나 **다른 도구**로 전환 |

### 2) 화면이 PRD대로 안 나올 때

대부분 PRD가 모호한 게 원인. 다음 순서로 점검:

1. **PRD를 ChatGPT/Claude에 다시 던져 자체 검증** — "이걸 보고 만들면 어떤 화면이 나올지 한 문장으로" 물어보기
2. AI 답이 본인 의도와 다르면 그 차이가 곧 PRD가 비어 있는 부분
3. **그 부분을 한 줄로 PRD에 추가** → Lovable에 다시 던지기

### 3) 도저히 본인 아이디어로 안 풀릴 때

본인 PRD를 포기하고 **백업 PRD 5개 중 하나**를 그대로 사용하세요. ([내 앱 컨셉 정의서 — 백업 PRD 5개](/day2/concept))

> **부끄러워할 필요 전혀 없습니다.** 강의 목표는 "독창적인 아이디어 생산"이 아니라 **"워크플로우를 손에 익히는 것"** 입니다. 백업 PRD로 흐름을 끝까지 가본 사람이, 본인 PRD로 중간에 멈춘 사람보다 더 많이 배웁니다.

## 진도가 빠른 사람 — 보너스 미션

화면 4~6개 + 빈/로딩/에러 다 끝났으면 다음을 시도하세요.

### 보너스 1 — 다크모드 토글

```text filename="다크모드 추가 프롬프트"
앱 우측 상단에 다크모드 토글 버튼을 추가해 줘.
- 해/달 아이콘
- 클릭 시 화면 전체가 다크 테마로 전환
- 사용자 선택은 localStorage에 저장 (새로고침해도 유지)
- 다크 테마 색상: 배경 #0f0f0f, 텍스트 #eaeaea, 카드 #1a1a1a
```

### 보너스 2 — 검색·필터 추가

```text filename="검색 기능 추가"
홈 화면 상단에 검색창을 추가해 줘.
- placeholder: "[도메인]을 검색해 보세요"
- 입력하는 즉시 목록이 실시간으로 필터링됨 (debounce 200ms)
- 검색어 X 버튼으로 초기화
- 결과 없으면 "검색 결과가 없어요" 빈 상태 표시
```

### 보너스 3 — 통계·요약 카드

```text filename="통계 카드 추가"
홈 화면 상단에 이번 주 요약 카드를 추가해 줘.
- 큰 숫자: 이번 주 총 [기록 개수]
- 부가 정보: 지난 주 대비 증감 (예: "+3 vs 지난주")
- 색상: 증가는 초록, 감소는 빨강, 동일은 회색
```

### 보너스 4 — 다른 도구로 같은 PRD 시도

같은 PRD를 [v0.dev](https://v0.dev)와 [bolt.new](https://bolt.new)에도 던져 보세요. **같은 명세에서 도구마다 결과가 어떻게 다른지** 비교하면 인사이트가 큽니다.

### 보너스 5 — 친구·가족에게 폰으로 보여주기

본인 폰의 Lovable 공유 URL을 가족·친구한테 보여 주고 30초 안에 무엇을 하는지 관찰. 사용자가 헤매는 부분이 곧 다음 PRD v3 의 재료입니다.
