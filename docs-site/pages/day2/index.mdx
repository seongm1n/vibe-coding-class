# Day 2 — 앱 컨셉과 첫 화면

오늘은 **본인 앱을 처음으로 띄우는 날**입니다. 어제 가져온 아이디어를 PRD라는 형태로 정리하고, 그걸 Lovable에 넣어 첫 화면을 만듭니다.

## 오늘의 학습 목표

- 본인 앱 컨셉을 **한 문장**으로 정의할 수 있습니다.
- ChatGPT로 PRD(요구사항 문서)를 만들고, **Lovable에 그대로 던질 수 있는 프롬프트**로 변환합니다.
- Lovable로 **첫 화면 + 최소 1개 서브 화면**을 띄웁니다.
- 무료 메시지 한도 안에서 **효율적으로 수정 요청하는 법**을 익힙니다.

## 늦게 오신 분 / 어제 못 들은 분 — 첫 챕터에서 따라잡기

- 첫 챕터 [아이디어 점검](/day2/idea-check)이 **늦은 분 따라잡기 + 아이디어 다듬기** 시간을 겸합니다
- 아이디어 안 가져오신 분은 거기서 주변에서 찾는 5가지 질문으로 1개 발굴
- 그래도 안 떠오르면 **백업 PRD 5개** 그대로 사용 OK

## 오늘 챙겨 갈 핵심 개념

### PRD = 프롬프트의 본체

- 비전공자도 PRD를 모르는 게 아니라 **이름만 모르는 것**입니다.
- "누가, 왜, 무엇을, 어떻게" 4개만 정리해도 충분한 PRD가 됩니다.
- AI에게 시킬 때 이걸 빼먹으면 결과가 일관성 없게 나옵니다.

### 프롬프트 작성의 3가지 원칙

1. **컨텍스트**: "나는 30명 규모 헬스장을 운영하는 사장이다"
2. **목표**: "회원이 출석 체크를 셀프로 할 수 있는 화면이 필요하다"
3. **제약**: "모바일에서 잘 보여야 하고, 글자는 크게, 한국어로"

### Lovable 메시지 아껴쓰기

- **무료 한도: 하루 5개 / 월 30개** → **한 번에 많이 시키고 결과 보고 다음 메시지** 패턴
- "조금씩 고치기"보다 "한꺼번에 큰 변화 요청"이 효율적입니다.
- 화면 단위로 끊어서 진행 (한 메시지에 한 화면 완성 목표).
- 자세한 한도 표는 [Lovable로 첫 화면 만들기](/day2/lovable-first-screen) 시작 부분에 있습니다.
