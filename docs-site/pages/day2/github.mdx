# GitHub 빠른 이해

내일(Day 3) 본격적으로 GitHub를 씁니다. 그 전에 **이게 뭐고 왜 필요한지**만 5분 안에 정리하고 갑시다.

## 한 줄 요약

> **GitHub = 코드를 보관하는 클라우드.**
> 구글 드라이브가 파일을 보관한다면, GitHub는 **코드를 보관**합니다.

## 왜 알아야 하나요?

내일(Day 3) 두 가지에 GitHub를 씁니다.

### 1. Supabase·Vercel 가입을 한 번에 끝내기

```
이메일로 따로 가입             GitHub로 통일
─────────────────              ─────────────────
Supabase: 이메일+비번  →       Supabase: "GitHub로 로그인" 클릭
Vercel: 이메일+비번    →       Vercel: "GitHub로 로그인" 클릭
                                ↓
비밀번호 3개 외우기            GitHub 한 번으로 끝
```

**GitHub 계정 하나가 다른 서비스 가입을 대신해 줍니다.** 비밀번호 외울 일이 줄어듭니다.

### 2. Lovable이 만든 코드를 자동으로 보관

Lovable에서 "GitHub 연결" 버튼만 누르면, **본인이 코드를 옮기지 않아도 GitHub에 자동 저장**됩니다.

```
어제까지: Lovable 안에 코드가 있음 (Lovable 계정 잃으면 끝)
오늘부터: Lovable + GitHub 양쪽에 코드 보관 (안전)
```

컴퓨터가 망가져도, Lovable 계정이 막혀도, **코드는 GitHub에 살아 있습니다**.

## 핵심 단어 3개 — 단어만 들어 두기

| 용어 | 한 줄 비유 |
|------|-----------|
| **저장소** (Repository, 줄여서 Repo) | 폴더 한 개. **앱 1개 = 저장소 1개** |
| **커밋** (Commit) | "여기까지 만든 걸 일단 저장" 버튼 |
| **푸시** (Push) | 저장한 걸 **GitHub 클라우드에 올리기** |

<Callout type="info">
**오늘은 단어만 익숙해지면 됩니다.** 내일 강사가 같이 누릅니다. 외울 필요 없습니다.
</Callout>

## 자주 나오는 오해

<Callout type="error">
**"개발자만 쓰는 곳 아닌가요?"**
아닙니다. 그냥 **클라우드 저장소**입니다. 회사 문서도, 책 원고도, 강의 자료도 GitHub에 올립니다.
</Callout>

<Callout type="error">
**"유료인가요?"**
무료입니다. 우리가 4일 동안 할 일은 전부 무료에서 됩니다.
</Callout>

<Callout type="error">
**"명령어를 외워야 하나요?"**
오늘은 No. Lovable이 자동으로 처리하는 부분이 대부분입니다. **버튼 한두 개**만 누르면 됩니다.
</Callout>

## 오늘 할 일 — 가입 1번

가입만 하면 끝입니다.

1. [github.com/join](https://github.com/join) 접속
2. 이메일·아이디·비밀번호 입력
3. 이메일 인증 → 끝 (약 2분)

가입 후 화면에서 **뭐 만지지 마세요.** 내일 같이 합니다.

<Callout type="info">
**GitHub 아이디는 본인의 영구 ID입니다.** 나중에 이력서·포트폴리오·채용에도 노출될 수 있으니 **짧고 읽기 쉽게** 만드세요.

- ✅ 좋은 예: `theo-park`, `minji2026`, `seongmin-kim`
- ❌ 나쁜 예: `xX_dragon99_Xx`, `qwerasdf123`, `abcdefg`
</Callout>

## 그래서 4일 강의에서 GitHub의 위치

```
Day 1: AI 도구 체험            (GitHub 없음)
Day 2: 화면 만들기              (GitHub 가입만)
Day 3: 데이터·인증·배포          ← GitHub 본격 등장
       ├─ Supabase 가입 (GitHub로)
       ├─ Vercel 가입 (GitHub로)
       └─ Lovable → GitHub 코드 연결
Day 4: 수익화·발표              (배포된 본인 앱이 GitHub에서 굴러감)
```

**Day 3에서 GitHub가 "허브" 역할**을 합니다. 그래서 오늘 가입을 미리 끝내야 내일 30분이 안 날아갑니다.
